> 版本：V1.0  
> 对应产品文档：`PRD_回响_完整版_V1.2.md` §5.1–§5.9、§8.1  
> 美术方向：**轻水彩浮雕地图为主体 + 象牙白微缩沙盘的材质克制 + 纸雕博物志的信息秩序**  
> 用途：供逐项生成、审核和回传原始美术素材；不是最终前端实现说明。

## 1. 先读：这份文档怎么用

文档一次性列出全部 Prompt，但**不要一次性无审核地跑完**。后续任务会引用前面已经通过审核的图片；如果风格母版没有锁定就继续生成，三层地图会逐渐变成三套画风。

推荐执行顺序：

1. 生成 `S00`，确认视角、材质、光照和调色。
2. 生成 `S01–S08`，确认世界、区域、群岛、状态、移动端构图和转场。
3. 生成 `W01–W07`，交付世界地图生产素材。
4. 生成 `R01–R07`，交付三张区域地图和四个合集地标。
5. 生成 `I01–I08`，交付三类群岛环境、九座视频岛和状态素材。
6. 生成 `C01–C05`，补齐通用地形、环境、回响与图形语言资产。

### 三道审核闸门

| 闸门 | 审核图片 | 通过条件 | 未通过时 |
|---|---|---|---|
| G1 · 风格锁定 | `S00–S01` | 明亮、轻盈、统一视角；不是儿童卡通、暗黑奇幻或房地产沙盘 | 只重做 `S00/S01`，不要继续 |
| G2 · 语义锁定 | `S02–S05` | 隐藏文字后仍能大致分辨经济/历史/科技；区域与群岛层级明显不同 | 调整地貌与地标，不靠加文字补救 |
| G3 · 状态锁定 | `S06–S08` | 金色只表示回响；未观看、最近新增、选中、内容丰富可以区分；移动构图与转场成立 | 调整状态层，不改变基础岛体画风 |

### 每次生成时的参考图传递规则

- `S00`：附用户提供的参考图。
- `S01–S08`：同时附用户参考图和已通过的 `S00_style_dna_approved.png`。
- `W` 组：附 `S00` 与 `S01`；分区提取任务额外附 `W02`。
- `R` 组：附 `S00` 与对应的 `S02/S03/S04`。
- `I` 组：附 `S00` 与 `S05`；某个合集的岛屿组额外附相应类别的群岛环境底图。
- `C` 组：至少附 `S00`；回响效果额外附 `S06`。
- 生成工具只能接收一张参考图时，优先使用最近一张已通过审核的母版，而不是最初参考图。

### 文件回传要求

- 回传原始 PNG；有透明通道时保留 Alpha，不要截图、转 JPEG 或经聊天软件二次压缩。
- 记录生成工具、模型版本、Prompt、seed（若工具支持）和输出尺寸。
- 一个任务对应一个文件；多版候选用 `_a/_b/_c`，通过后改名 `_approved`。
- 透明背景不稳定时，使用均匀纯色背景 `#F5F3EC`，不要添加落地台、相框或摄影棚地面。
- 不提前锐化、降噪、抠图或放大；这些由后续资产处理统一完成。

推荐文件结构：

```text
map-art-source/
├─ 00_style/
├─ 10_world/
├─ 20_region/
├─ 30_archipelago/
└─ 40_common/
```

## 2. 全局视觉规范

### 2.1 视觉 DNA

- 正交投影或极弱透视、无明显消失点；相机方位固定，俯角相对水平面约 55°（即离垂直俯视约 35°）。所有任务以已批准 `S00` 的实际视角为最高准则，数字只用于防止模型误读。
- 地形顶面：克制的水彩、水粉和彩铅微细节，不出现脏重纸纹。
- 地形侧面：象牙白、浅砂色的细腻石膏/雪花石膏微缩模型质感。
- 光源：左上或正上偏左的柔和日光；阴影浅、软、短，不做黑色厚阴影。
- 海洋：通透浅青蓝，浅滩略亮；不做深海、暴风或高反射写实水面。
- 构图：世界是一块连贯大陆；区域是一张完整场景；只有群岛层明显拆成多座岛。
- 页面气质：明亮、安静、通透、编辑感、高级内容产品、轻游戏化。

### 2.2 固定调色角色

| 角色 | 参考色 | 使用规则 |
|---|---|---|
| 页面暖白 | `#FAF9F6` | 页面与空白背景 |
| 浅海 | `#BFE7EA` | 地图主海域 |
| 深一点的浅海 | `#8FD3DA` | 岸外层次，不能变深蓝 |
| 象牙地表 | `#EFE4CC` | 通用土壤、侧面与中性地形 |
| 经济 | `#DDD9A6` / `#E7CFA8` | 暖麦色、浅鼠尾草、浅陶土；不是“生态绿” |
| 历史 | `#D8C2D3` / `#DCBDAE` | 灰紫、赭粉、旧石色 |
| 科技 | `#BED7DD` / `#CEC6E1` | 雾蓝、浅紫、冷灰；禁止霓虹 |
| 墨色 | `#57534A` | 普通图标、道路和结构线 |
| 回响金 | `#D9A93F` | **只用于回响**，不用于普通道路、边界、新增或类别装饰 |

### 2.3 类别语义

| 类别 | 必须出现的识别线索 | 禁止误读 |
|---|---|---|
| 经济（代码 ID `eco`） | 三角洲、贸易港、城市群、中央塔楼、放射道路、市场/生产网络 | 不能画成环保、森林或自然生态主题 |
| 历史（`his`） | 古道、城墙、遗迹、档案馆、考古层理、旧河道 | 不能只靠一本书或一座普通宫殿 |
| 科技（`tech`） | 研究所、观测塔、规则桥梁、精密网络、知识港口 | 不能变成赛博城市、宇宙基地或蓝色霓虹 HUD |

### 2.4 全局负向约束

除 `C04` 回响效果任务外，所有 Prompt 都默认包含以下约束：

```text
不要生成任何文字、汉字、英文字母、数字、Logo、按钮、输入框、标签或 UI；不要金色道路、金色边界或金色普通建筑；不要粗白色行政边界；不要深色/近黑背景；不要暗黑奇幻、史诗战争、赛博朋克、霓虹、科幻 HUD、宇宙、浮空岩盘；不要真实卫星地图、航拍照片、写实城市；不要儿童卡通、糖果色、塑料玩具、积木式 low-poly；不要厚重黑影、强景深、移轴摄影模糊、镜头炫光、体积云遮挡主体；不要随机改变相机角度、光源方向或物体比例；不要裁切主要对象，不要重复建筑和畸形结构。
```

### 2.5 尺度、基线与分层规则

- 所有独立资产使用 `S00_approved` 的正交相机和同一光源，不凭文字里的角度重新猜视角。
- 资产板用于先审造型；通过后，最终交付应做到“一对象一文件”，单对象长边至少 1536px，重要地标/岛体建议 2048px。
- 单对象在方形源画布中占宽约 68%–76%，四周留透明安全区；不要通过后期放大超过 1.25 倍，否则笔触和纹理密度会失真。
- 区域地图中的普通合集地标建议占场景宽 14%–18%；`c1` 裂痕城市和 `c4` 知识港可达 18%–22%；`c2` 塔楼主体更窄但可稍高。
- 群岛地图中的普通视频岛建议占场景宽 18%–21%；内容丰富的 `v1` 可达 23%–25%；较小的 `v3` 建议 16%–18%。最高建筑不超过岛体可见宽度的约 35%。
- 独立资产的对象本体与投射阴影/AO 最好分别交付两个 Alpha 文件；工具无法分层时保留原始无背景图，不把阴影烘焙进暖白底。
- 桌面与移动端必须从同一批准母版派生安全构图和 LOD；移动端不能重新生成一张位置、轮廓和语义漂移的新地图。

### 2.6 多对象资产板的单对象输出模板

`W06`、`I04–I07`、`C01–C04` 先以资产板审核。资产板通过后，对其中每个对象重复执行下面的编辑模板；无需重新发明 Prompt。

```text
以所附已批准资产板为唯一造型和风格真源，只提取并高分辨率重制其中的【对象名称】。禁止重新设计轮廓、建筑布局、相机、比例、颜色或光照；保持正交投影、无消失点、与 S00_approved 完全相同的视角和左上柔光。输出单个对象，居中，占方形画布宽约 68%–76%，四周留足透明安全区，长边至少 1536px（重要地标 2048px），透明 PNG。保留对象自身细节；对象本体与柔和 AO/投影阴影若工具支持则分别输出。不要生成其他对象、文字、UI、底座、背景或额外装饰；除对象明确属于“回响效果”外，不得使用金色。
```

## 5. S 组：风格与视觉审核

### S00 · 统一视觉 DNA 母版

- **用途**：锁定三个层级共享的视角、材质、光照、色彩和细节密度。
- **性质**：仅供审核。
- **参考输入**：用户提供的 Aerolith 风格参考图。
- **建议输出**：16:9，长边至少 2048px，PNG。
- **文件名**：`00_style/S00_style_dna_v01.png`

```text
制作一张无文字的高端知识地图视觉 DNA 母版，在同一张宽幅画面中用三个彼此留有暖白空隙的无边框场景，依次展示：一块连贯世界大陆的局部、一张更密集的区域地图、一组轻盈群岛。统一使用约 30 度正交俯视、左上柔和日光、明亮浅青海洋。地形顶面是克制精细的 watercolor gouache 与 colored-pencil 微细节，侧面是薄而精致的 ivory alabaster miniature model 质感，柔和短阴影、低饱和粉彩。世界片段呈现经济/历史/科技三种语义地貌；区域片段呈现道路、河流与可识别地标；群岛片段呈现浅滩、岛体与少量环境云。整体 airy, bright, elegant, editorial, premium, refined light-fantasy product map。没有页面 UI，没有文字，没有金色元素。

遵守全局负向约束，尤其禁止粗白边、儿童卡通、厚重浮空岩盘、暗黑奇幻、赛博霓虹和真实卫星地图。
```

**验收点**：三个层级像同一世界；顶面有水彩温度、侧面有克制实体感；静止画面已经成立；不存在“生态大陆”的误导。

### S01 · 世界地图完整视觉基准

- **用途**：确定首页地图的连贯大陆轮廓、三类别位置和页面留白比例。
- **性质**：仅供审核，后续 `W02` 的构图参考。
- **参考输入**：用户参考图 + 已通过的 `S00`。
- **建议输出**：16:10，3200×2000 或最接近的高分辨率，PNG。
- **文件名**：`00_style/S01_world_keyframe_v01.png`

```text
一张明亮高端的 2.5D 知识世界地图完整关键视觉，16:10 横向构图，画面上方约 26% 保持干净暖白和极轻云雾留白，不生成 Logo、输入框或任何 UI；地图主体位于中下部。世界是一块自然连贯的大陆，不是三块分离浮岛，由山脉、河流、植被、道路密度和轻微色差自然分成三个区域。西部或左侧为“经济”语义：暖麦色与浅鼠尾草色的贸易三角洲、港口、城市群、中央塔楼和放射道路；南部或中下方为“历史”语义：灰紫与赭粉丘陵、古道、城墙、遗迹、档案馆和考古层理；东北或右侧为“科技”语义：雾蓝浅紫高地、研究所、观测塔、规则桥梁与精密网络，但不赛博。浅青海洋、精细浅滩、小型山脉湖泊、边缘轻云。30 度正交俯视，顶面为克制水彩水粉和彩铅细节，海岸侧面薄而像象牙白微缩沙盘，左上柔光，低饱和、通透、安静、高级。普通道路和地标使用灰墨与类别色，完全不使用金色。

不要生成文字、符号说明、UI、粗白色区域线或行政拼图感；遵守全局负向约束。
```

**验收点**：世界整体连贯；三个区域不看文字仍能大致猜出经济/历史/科技；上方留白足够放真实 DOM Hero；普通元素没有金色。

### S02 · 经济区域地图视觉基准

- **用途**：锁定经济区域页的完整场景和两个合集的空间关系。
- **性质**：仅供审核，后续 `R01/R04/R05` 的母版。
- **参考输入**：`S00` + `S01`。
- **建议输出**：16:10，长边至少 2560px，PNG。
- **文件名**：`00_style/S02_region_economy_keyframe_v01.png`

```text
进入“经济大陆”后的独立区域地图关键视觉，仍是约 30 度正交俯视和相同左上柔光，但细节密度比世界地图提高一倍。完整区域是一片暖麦色、浅鼠尾草与浅陶土色交织的贸易盆地和沿海三角洲，有河网、桥梁、港湾、梯田、城市街区、仓储与市场道路，强调流通、生产、分配和秩序，不表现自然生态保护主题。场景内有两个清晰但不贴文字的主地标：一座带有克制地表裂痕、分区差异明显的城市，代表“看懂通胀这门课”；一座轴线规整、比例修长的中央塔楼和环状道路，代表“宏观经济 30 讲”。两地由普通灰墨/类别色道路与河流连接，不使用金色。浅青水面、柔和象牙侧边、精细水彩地貌、轻沙盘实体感，明亮高级，不拥挤成游戏城建。

不生成文字、数字、货币符号、金币、股票 K 线、绿色环保图标或金色道路；遵守全局负向约束。
```

**验收点**：经济而非生态；裂痕城市和中央塔楼一眼可区分；两个地标属于同一完整场景。

### S03 · 历史区域地图视觉基准

- **建议输出**：16:10，长边至少 2560px，PNG。
- **文件名**：`00_style/S03_region_history_keyframe_v01.png`

```text
进入“历史大陆”后的独立区域地图关键视觉，约 30 度正交俯视，左上柔光，明亮的灰紫、赭粉、旧石与浅砂色。完整场景是一片被时间层层沉积的丘陵与河谷：古道、断续城墙、旧河道、石桥、考古剖面、档案馆式建筑、残留聚落和少量树木。主地标是一片有明确层级与中央识别点的古老遗迹群，代表“三国的另一种读法”；遗迹应像被重新阅读和发掘的历史现场，不是阴森废墟，不直接复刻具体古战场。地表保留克制水彩与彩铅纹理，侧面为薄象牙沙盘质感，薄雾只在远边，不遮挡主体。道路和结构线用灰墨或旧石色，没有金色。
```

## 11. 单张图片验收表

每次生成后按下表检查。任何一条“红线”失败都应重做，不要靠后期小修掩盖方向错误。

### 11.1 构图与技术

- [ ] 相机约 30 度正交/弱透视，与母版一致。
- [ ] 光源仍在左上或正上偏左，阴影柔和、短、浅。
- [ ] 主对象完整，没有裁切、贴边或被云遮挡。
- [ ] 透明资产四周有安全留白，没有背景残留或白边。
- [ ] 同一资产组比例统一，没有一个对象突然采用另一视角。
- [ ] 需要同画布对齐的图与母版像素坐标一致。

### 11.2 产品语义

- [ ] `eco` 被明确表现为经济，不是生态。
- [ ] 经济/历史/科技不只靠颜色区分，地貌和主地标也不同。
- [ ] 世界是一块连贯世界，不是三块分离卡片或浮岛。
- [ ] 区域边界主要由自然地势、道路、河流和材质表达，无粗白边。
- [ ] 地图没有文字、数字、Logo、按钮或信息卡。
- [ ] 普通道路、地标、选中和最近新增没有使用回响金。

### 11.3 状态

- [ ] 未观看 = 降对比/薄雾，不等于锁定或消失。
- [ ] 最近新增 = 非金色生长/搭建痕迹，且仍正常可见。
- [ ] 有回响 = 唯一暖金信标、光路或呼吸。
- [ ] 当前选中 = 抬升、阴影和水面反馈，不冒用金色。
- [ ] 内容丰富 = 尺度或建筑密度适度增加。
- [ ] 状态可叠加，特别是 `v3` 的“未观看 + 有回响”。

### 11.4 美术红线

- [ ] 不像真实卫星地图或自然照片。
- [ ] 不像儿童卡通、糖果色玩具或积木式 low-poly 游戏。
- [ ] 不像暗色科幻、宇宙、战术沙盘或霓虹赛博界面。
- [ ] 没有厚重悬浮岩盘、大片黑影、强体积云或灾难感。
- [ ] 静止画面已经完整，不依赖动画掩盖美术不足。
- [ ] 地图仍服务于知识导航，不压过页面真实输入动作和信息内容。

## 12. 失败重试规则

1. **风格漂移**：始终重新附 `S00_approved`，不要拿失败图继续做下一张参考。
2. **经济被画成生态**：明确加入 `economics, trade, markets, production and distribution`，删除 `eco` 缩写，禁止叶子/环保/绿色能源主符号。
3. **科技变赛博**：删除 glow、futuristic、cyber 等词，改用 `precision, research infrastructure, modular work systems, quiet engineering`。
4. **历史变战争游戏**：删除 battle、empire、epic 等词，改用 `archive, excavation, layered evidence, old roads, reinterpretation`。
5. **侧面太厚**：强调 `thin 6–10px visual relief, not floating slabs`，减少 extrusion、floating island 等词。
6. **水彩太脏**：强调 `restrained clean watercolor wash, very subtle paper grain, crisp small landmarks`。
7. **透明图边缘错误**：不要反复让模型修边；交回带纯色背景的原图，由后续统一抠图。
8. **同画布提取发生变形**：立即停止生成 `W03–W05`，只交 `W02`，由人工建立蒙版。
9. **普通元素误用金色**：直接重做，不能靠后期整体去色，因为会破坏回响的颜色语义。
10. **对象过密**：优先删飞鸟、船、云、装饰树和微建筑，不删类别主地标与交互安全区。

## 14. 最终交付定义

完成本手册不等于“拿到 36 张图就结束”。最终地图资产包应由以下内容组成：

- 一张世界环境底图、一张世界母版、三个独立类别区域层。
- 三张区域底图、四个独立合集地标。
- 三类群岛环境底图、九个独立视频岛体。
- 可组合的未观看、最近新增、选中、有回响和悬浮状态层。
- 可复用地形、建筑、云雾、水纹与回响效果模块。
- 独立 SVG 视觉轮廓、命中代理、标签锚点和相机目标。
- 桌面与移动端构图/LOD 版本。
- 原始无损母版、运行时压缩资源和完整来源记录。

只有当这些资源能够被 `MapScene / MapItem` 配置独立引用、状态可以组合、三个层级保持同一视觉世界，并通过移动端命中与性能检查时，才算完成地图美术生产。
