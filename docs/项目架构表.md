# 知音 · 项目架构表

用途：快速定位"某个功能/页面/逻辑该改哪个文件"。按"生产代码 → 解析管线 → 原型 → 文档"排列，越靠前越是当前主战场。

## 一、生产代码 `web/`（Next.js 16 + React 19，正式产品）

跑法：`cd web && npm run dev`（端口 3100）

### 4. 数据与配置 `web/lib/`

| 文件 | 作用 | 常改动点 |
|------|------|----------|
| `data.ts` (754行) | 种子数据（预烘焙兜底/演示）。地图上有真实解析内容时让位，详情页仍可直链访问 | 改种子内容/数据结构 |
| `map-config.ts` | 地图布局配置（大陆/群岛坐标、样式参数）；`createMapItem` 供真实数据动态生成地图项 | 调整地图排布、新增分类/群岛 |

| `server/pipeline.ts` | 解析管线：直链→下载→ASR→瘦脉络→回响/known→认知拓展→AI分类→归入合集 | 改 AI 契约、加管线步骤 |
| `server/real-data.ts` | 真实解析数据 → 地图三层可渲染形状（槽位坐标按入库顺序固定分配） | 改真实数据的地图呈现 |

### 5. 美术资源边界

|---|---|---|

## 二、解析管线测评 `eval/` + `scripts/`（独立零依赖，验证"链接→解析→ASR"通路）

跑法：根目录 `npm run dev`（自动拉起抖音解析 sidecar + eval 服务）或 `npm run server`（仅 eval 服务，端口 6060）

| 文件 | 作用 | 常改动点 |
|------|------|----------|
| `eval/server.mjs` (498行) | 管线服务端：解析链接→下载→抽音频→火山豆包 ASR | 解析源适配、ASR 调用参数 |
| `eval/index.html` | 管线测试页面 | 测试用简易前端 |
| `eval/.env` / `.env.example` | ASR key、sidecar 地址等环境变量 | 换 key、改 sidecar 地址 |
| `scripts/dev.mjs` (93行) | 一键启动脚本：拉起 docker sidecar + eval + web | 启动流程、sidecar 健康检查 |
| `scripts/mix_enum.py` (61行) | 辅助脚本（枚举/混合处理，Python） | 视情况改 |

依赖：Node 20+、ffmpeg、抖音解析用 `wujunwei928/parse-video` docker sidecar、火山豆包 ASR key。
