# 世界地图动效方案：调研与诊断

*生成：2026-07-15 | 范围：世界地图首页（`WorldMapStage` + `WorldMapCanvas` 的 R3F 正交场景）*
*配套：`docs/调研_网页地图交互最佳实践.md` 只覆盖 pan/zoom 相机手感，本文补齐它遗漏的「视觉动效 / 氛围动画」一层。*

## 结论先行

1. **美术资源足够，甚至有富余**——多套已产出的资产（三区地标、氛围叠加层、水面细节、4 个备用云变体、4 个备用涟漪变体）在场景里**根本没有被渲染**。当前动效差不是缺素材，是没把素材用上、动作太胆怯。
2. **最像"贴图拼贴"的根因是假视差**：正交相机下 z 坐标只决定渲染顺序、不产生视差，所以三层云是跟地形 1:1 平移的，永远"贴"在画面上。
3. reactbits / motion.dev 那类"高级感"不是靠加更多特效堆出来的，而是靠**入场编排、真实景深、有分量的反馈、克制的常驻呼吸**。下面给出按优先级排好的可落地清单，并标注每一项对应 reactbits/motion 的哪个惯用手法。

---

## 一、当前场景已有什么（先讲清楚现状，避免重复造轮子）

`WorldMapCanvas.tsx` 里实际在跑的动效：

| 动效 | 实现 | 现状问题 |
|---|---|---|
| 相机阻尼 | `maath` `easing.damp` 0.2 平滑 x/y/zoom | ✅ 手感 OK，唯一合格项 |
| 水面 shader | flow + glint，`mix(paperSea, water, 0.20)`、glint `*0.065` | 位移/高光被压到几乎不可见，采样 4 张贴图却几乎看不到动 |
| 漂移云 | far/mid/front 三层，正弦上下浮 + 线性横移 | **假视差**：正交下三层同速平移，像贴纸滑过 |
| 区域辉光 | mask 贴图 additive，hover 0.09 / selected 0.15 | opacity 太低，hover 几乎无反馈 |
| 涟漪爆发 | 点击区域时两个同心环扩散 1.55s | ✅ 是唯一有"分量"的反馈，但只在点击时出现 |
| DOM 标签 | CSS hover `translateY(-3px)` 220ms | 反馈全靠它撑，孤木难支 |
| 入场 | `sceneArrive` 整块 0.7s fade+位移 | 整张 webp 一次性淡入，没有"世界在生成"的层次 |

## 二、被浪费的美术资源（这是"够不够"的直接答案）

以下资产**已在磁盘 / manifest 中，但 `WorldMapCanvas` 从未渲染**：

- `regions[].landmark` —— `world_landmark_{economy,technology,history}_lod1_v01.webp`，**三区地标独立图完全没用**。地标目前是烘焙进 `world_terrain` 母图的死像素，无法单独发光/呼吸/浮动。
- `assets.environment` —— `world_atmosphere_overlay_lod1_v02.webp`，氛围叠加层没进场景（现在只有 CSS 的 `worldMapStage__ambient` 渐变代偿）。
- `world/effects/world_water_detail_lod1_v01.webp` —— 水面细节层，磁盘上有，manifest 都没登记。
- `assets.water.ripples[2..5]` —— `arc-wide / arc-low / wake / foam-patch` 四个涟漪变体只用了前两个。
- `clouds.{far,mid,front}` 各 **4 个变体只各用了 1 个**（`CLOUD_CONFIGS` 里 far-1/2/3 复用 far[0..2]，mid 用 2 个，front 只用 1 个）。

**判断：不需要再找美术产新图。** 先把上面这些接进去，动效的"信息量"立刻翻倍；真要补，只需要极少量（见第四节）。

## 三、推荐动效清单（对应 reactbits / motion 手法）

按"改动小、收益大"排序。分三档：**修（现有的先救活）→ 接（把废弃资产用起来）→ 加（新做少量）**。

### A. 修——让已有动效有分量（0 新资产）

1. **真视差**（对应 reactbits `Threads`/`Waves` 那种"图层各自运动"的层次感，motion 的 `useTransform` 视差）
   正交相机下手动做视差：云层位置 = `相机delta × 视差系数`（far ≈0.15、mid ≈0.4、front ≈0.75），而不是让它们待在世界坐标里被相机 1:1 拖走。拖动/缩放时前景云比大陆跑得快，"贴纸感"立刻消失。**这是性价比最高的一条。**

2. **把区域反馈做出分量**（对应 reactbits `Spotlight Card` / `Star Border` 的"选中即点亮"）
   - hover 辉光 0.09 → 0.28~0.35，并给一个 120ms 的快速 attack、缓慢 release（现在是对称 damp，读起来发闷）。
   - selected 时叠加一圈冷白海岸光（设计语言里"选中用冷白海岸光与浅青水波"），不是只调一个 additive 透明度。

3. **水面救活**（对应 reactbits `Silk`/`Iridescence` 的低频流动）
   `mix` 0.20 → 0.32，glint `*0.065 → *0.11`，flow 位移速度提约 40%。仍然"低频低对比"（守设计语言），但要能被看见。移动端维持低值。

4. **入场编排**（对应 motion 的 `stagger` + reactbits `Animated Content`）
   别再整张 webp 一次性淡入。分 3~4 拍：海面/远云先亮 → 大陆从底部微微升起（scale 0.98→1 + Y 位移）→ 三区地标依次点亮 → 标签最后浮现。世界"长出来"的感觉，是这类站点高级感的主来源。

### B. 接——把废弃资产接进场景（低成本，中高收益）

5. **地标独立成层 + 常驻微动**（对应 reactbits `Meta Balls`/`Orb` 的"焦点在呼吸"）
   把三张 `landmark` webp 从母图里抽出来单独渲染（母图对应位置需留空或压暗，见第四节）。独立后可以：
   - 常驻极缓慢的呼吸（scale ±0.8%、8~10s），让区域中心"活着"；
   - hover 时该区地标轻微上浮 + 投影加深（motion `whileHover` 的空间反馈）；
   - Echo 区地标带暖金微光（守"暖金只属于回响"）。

6. **氛围叠加层进场景**（对应 reactbits `Aurora`/`Dark Veil` 的顶层辉光罩）
   用 `world_atmosphere_overlay` 替代 / 叠加现在的 CSS 渐变，作为最上层的柔光，缩放时轻微反向位移制造空气厚度。

7. **涟漪变体轮换 + 水面细节层**
   点击不同区域时从 `ring/arc/wake/foam` 里轮换，不要每次都同两个环；把 `world_water_detail` 作为水面之上的一层缓动 glaze，进一步压掉"塑料水"观感。

### C. 加——少量新动效（可选，锦上添花）

8. **指针光斑 / 磁吸**（对应 reactbits `Spotlight` / `Magnet`、motion `whileHover` 弹簧）
   标签靠近指针时轻微磁吸位移，或大陆上跟随指针的一小片柔光。**克制**——设计语言反对"装饰过多淹没输入框"，这条要么很淡要么不做。

9. **首屏一次性引导动效**（对应 reactbits `Click Spark`）
   首次进入时在某个 Echo 区做一次涟漪，暗示"可点击"，只播一次。

> **红线（守 `world-map-design-language.md`）**：云不许遮输入框/标签/主地标；暖金只给回响；不加第四块大陆；`prefers-reduced-motion` 下全部降级为静止。以上每条都要过这几关。

## 四、真要补的美术（极少）

绝大多数靠"接现有资产"解决。只有 **A5 地标独立成层** 需要美术配合：

- 现在地标是烘焙进 `world_terrain` 母图的。要独立浮动，母图对应位置得**留空或压暗成底座**，否则会出现"母图里一个地标 + 浮层一个地标"的双影。
- 两条路：①请美术出一张"去地标版"母图 + 三张带透明通道的地标图（干净，推荐）；②工程侧用 region mask 在母图地标处压暗代偿（零美术，但边缘可能糊）。

**除此之外无需新图。**

## 五、建议的落地顺序

先做 **A1 真视差 + A2 反馈分量 + A4 入场编排**——零新资产，一个下午，观感提升最大。看效果后再决定是否投入 B5（地标独立，需美术）。

> 实施时按用户既定规矩：改版前先跑 dev 截当前基线图，改完再截对比图自检，美观项配示意图交用户裁决，不拿本文当"已批准"。

## 附：reactbits / motion 对照速查

| 想要的观感 | reactbits 参考组件 | motion 手法 |
|---|---|---|
| 图层各自运动的景深 | Threads, Waves, Silk | `useScroll`/`useTransform` 视差 |
| 选中即点亮 | Spotlight Card, Star Border, Glow | `whileHover`/`animate` variants |
| 低频流动氛围 | Silk, Iridescence, Aurora | shader / canvas 常驻 loop |
| 焦点在呼吸 | Orb, Meta Balls | 无限 `repeat` 弹簧 |
| 世界逐步长出 | Animated Content, Fade Content | `staggerChildren` + `AnimatePresence` |
| 指针磁吸/光斑 | Magnet, Spotlight, Click Spark | `whileHover` 弹簧 + 指针追踪 |
