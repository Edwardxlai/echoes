# 回响 · 讨论区设计方案 V1

> 状态：设计草案，待并入 PRD V1.2（拟作变更摘要 #16 + 展开现有 §6.5）
> 定位：把 PRD §6.5「讨论区 gated（愿景层）」从一句话愿景，展开为黑客松可落地的三层锚定 + 结构性门槛方案
> 日期：2026-07-15

---

## 〇、一句话方案

> **门槛用"数据结构"实现，不用"声望分数"实现。** 讨论沿知识地图三层锚定（板块 / 合集 / 节点）；能不能在某处发言，取决于"你的知识库里有没有对应的东西"，而不是"你攒了多少积分"。这既是黑客松最省的实现，也是回响相对普通评论区唯一抄不走的差异点。

---

## 一、为什么不做声望系统（黑客松红线）

普通做法是 Discourse / StackOverflow 式的信任等级或声望分：读够 → 发够 → 攒赞 → 逐级解锁权限，再叠一层"曝光随赞加权"。这套东西有两个问题，对黑客松都是致命的：

1. **重**：要建积分表、升级规则引擎、曝光加权算法，是一整个子系统，和"两周能 demo 出来的东西"不匹配。
2. **危险**：曝光随累计赞加权 = 马太效应发动机，头部会垄断，新声音被埋——这是用户明确担心、且我们没有运营人力去调平的坑。

**结论：整套"声望 + 曝光加权"在本期直接不做。** 回响手里有一个别人没有的东西——一张有语义结构的知识地图——足以用"结构性门槛"平替掉"声望门槛"，见下节。

---

## 二、核心机制：门槛 = 数据结构

讨论区分三层，每层的发言门槛不是"算出来的等级"，而是"一次数据库查询就能回答的事实"：**你的知识库里有没有对应的锚点。**

| 讨论层 | 锚定对象 | 对应地图层 | 发言门槛（结构性，运行时查询，非积分） |
|---|---|---|---|
| **板块广场** | 大类（经济/历史/科技） | 世界地图页 | 你在该板块解析过的资源数 ≥ N |
| **合集讨论区** | 一个合集（不含散的单视频） | 区域/群岛地图页 | 你已解锁/解析过该合集 |
| **节点讨论** | 单视频里的一个节点 | 视频解析页 | 见下方两种入口，**自动满足** |

节点讨论有两个入口，门槛都是"数据本身决定"，无需任何额外判定：

| 节点入口 | 议题种子 | 发言门槛（自动） | 你在跟谁聊 |
|---|---|---|---|
| **往深想**（§6.3.2 延伸） | AI 生成的开放题（无唯一答案） | 你解析过这条视频 | 所有解析过这条视频的人（较广） |
| **回响**（§6.2） | 你和他的知识库在此节点交汇 | 你已回响到这个节点 | 知识地图与你在此重叠的人（较窄、更精准） |

> **实现要点**：不建"用户等级"字段。发言权是一个运行时函数 `canWrite(user, discussion)`，它做的只是查"该 user 的观看史/回响关系里有没有这个 anchor"。整个门槛系统 = 三个 `EXISTS` 查询，没有积分表、没有升级 cron、没有曝光加权。

---

## 三、差异点：回响讨论为什么强过评论区

普通单视频评论区的病根是**路人性**——评论的人只是偶尔刷到，彼此没有共同上下文，所以聊不深、易口水。回响讨论用一道结构性准入同时解决两件事：

- **准入即领域投入证明**：能进来聊的人，都已用"亲手解析过回响到此节点的内容"证明了自己在这个领域有积累——等价于微信读书圈子"高门槛筛走低质量用户"的效果，但门槛是自动的，不靠人审。
- **同节点、异来路、共上下文**：所有参与者共享同一个知识点，且各自是**从不同合集/不同路径**抵达的。同一个点、不同的来路 = 天然的观点碰撞源。这正是评论区结构上给不了的东西。

这也顺势复用了回响已有的资产：
- 议题不用用户自己找——**AI 的"往深想"延伸题就是现成的高质量话题种子**，直接解决讨论区最难的"没人开第一贴"冷启动问题。
- 一条想法可以**引用另一个节点/合集作论据**（`quoteRef`，见 §六），是"回响关系词"的自然延伸——讨论本身也顺着知识地图的连接扩散，强化"知识变成一个世界"的核心隐喻。

## 四、门槛数值与冷启动

> 原则（来自 Discourse 官方校准经验）：**新社区门槛过高会直接扼杀讨论，初期定低，随规模再抬。**

| 层 | 初期（demo / 冷启动） | 成熟后 |
|---|---|---|
| 节点·回响讨论 | 结构自动（回响到即可） | 不变 |
| 节点·往深想讨论 | 结构自动（解析过该视频即可） | 不变 |
| 合集讨论区 | 解锁/解析该合集即可 | 不变 |
| 板块广场 | 该板块解析 ≥ 3 条 | 抬到 ≥ 5 条 |

- 只做 **两态**：游客（只读，全开放）/ 板块公民（可发帖+回帖）。**本期不做第三态"曝光加权"。**
- 已入场者**零版规、零格式要求**（学网易云"无为而治"：早年论坛用复杂版规追求精华，结果反而做死了社区）。门槛用于筛人，进来之后不再给表达设障碍。
- 发帖与回帖**同时解锁**，不拆成两道门。

## 五、头部垄断的替代方案（不做曝光加权，但让好内容能浮起来）

砍掉算法加权后，用三个"排序规则"（不是打分系统）让优质内容自然可见，且不制造马太效应：

1. **「最新 / 热门」双 Tab 并列**——新声音永远有一个不与头部竞争的入口（学微信读书圈子"最近"）。
2. **热门用时间衰减**（HN 式：分数 ÷ 时间^重力），不是累计赞。老帖再高赞也会自然沉下去，给新帖腾位置。
3. **热门是"每个锚点局部"的，不是全站的**——你在科技板块的高赞，在经济板块的讨论里毫无加成。天然把可见度限制在很小的范围内，与"按板块"思路一致。

这三条加起来约几十行排序逻辑，不拖累工期。真正的成长/曝光体系，等验证了讨论区有人用之后再说。

## 六、数据模型（最小集，接 PRD §8 风格）

```
Discussion(讨论线) {
  id
  scope:     "board" | "collection" | "node"      // 锚定层级
  anchorId                                          // bigCategoryId | collectionId | nodeId
  seedType:  "echo" | "extend" | "manual"           // 议题来源：回响 / 往深想 / 用户发起
  seedRef?                                          // extend→指向该延伸问题；echo→指向回响对
  title
  createdBy, createdAt
  postCount, participantCount
}

Post(帖 / 想法) {
  id, discussionId
  authorId
  body                                              // 纯文本，零格式要求
  quoteRef?                                         // 可引用另一个 node / collection 作论据（回响关系词延伸）
  parentPostId?                                     // 楼中楼
  likeCount, createdAt
}

// 门槛不落库为"等级"，是运行时查询：
canWrite(user, discussion) =
  scope=node       → 该 user 已回响到该 node（回响讨论）
                     或 已解析过该 node 所属 video（往深想讨论）
  scope=collection → 该 user 已解锁/解析该 collection
  scope=board      → 该 user 在该 bigCategory 下解析资源数 ≥ N
```

- 复用 §8 已有的 `bigCategoryId` / `collectionId` / 节点 id 和用户观看史，不新建平行的用户画像。
- `seedType=extend` 时，`seedRef` 指向 `CognitiveExpansion.extend[]` 里的那条问题——**AI 已经把话题准备好了**，`Discussion` 只是给它挂一个可回帖的容器。

## 七、与 PRD 现有条目的关系

- **展开 §6.5**：现有 §6.5 只定义了两条入口（①合集合成→发起讨论 ②往深想→进入同题讨论）和"门槛由观看史决定"一句话。本方案把它展开为三层锚定 + `canWrite` 结构性门槛，并**新增第三条入口：回响（§6.2）→ 就地发起/进入该节点的讨论**。
- **不冲突 §6.4.4 回响边界**：回响仍专指"本次内容 ↔ 用户历史观看史"的点对点关系，不改。这里只是把"两个人回响到同一节点"这一事实，额外用作**节点讨论的准入条件与议题种子**——是对回响的复用，不是给回响加第二套语义。
- **视觉呈现另行评审**：讨论区 UI 必须过 `design-language.md` v3.2 的"无卡片盒子堆砌"红线（不得逐帖套圆角+阴影盒子）。本方案**只定机制，不定视觉**；具体呈现（帖子怎么成"面"、如何沿用脉络点线/素色板语言）走设计评审，配示意图对比后再拍板。

## 八、黑客松落地范围（P0 / P1 / roadmap）

| 优先级 | 内容 | 理由 |
|---|---|---|
| **P0（最能体现差异化，门槛零开发）** | 节点讨论：往深想入口 + 回响入口；`canWrite` 结构判定；最新/热门双 Tab | 门槛是现成数据查询，且回响讨论是唯一抄不走的卖点，demo 高潮就在这 |
| **P1** | 合集讨论区（合集级锚定）；`quoteRef` 引用另一节点作论据 | 合集是天然话题边界，复用合集合成页；引用强化地图隐喻 |
| **Roadmap（本期只 seed + 占位）** | 板块广场（大类级）；任何曝光成长体系 | 大类级讨论需要跨合集内容密度，社区没到临界规模前先不做；曝光加权明确延后 |

> 沿用 §10.2 红线：讨论区不得反过来挤占回响/认知拓展的打磨时间。若排期冲突，讨论区让位。

3. **本期是否只上 P0（节点讨论）验证**，合集讨论区放 P1、板块广场放 roadmap？（本方案倾向是）
