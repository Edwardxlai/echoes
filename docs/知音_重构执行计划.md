# 知音 · 重构执行计划

> 状态：Phase 0 / Phase 2 已完成，Phase 1 并行实施中  
> 日期：2026-07-21  
> 配套：决策与细化见 `PRD_知音_完整版_V1.2.md` §十五（变更 #19）；逐条实施规格见 `知音_跨视频与多模态重构方案.md`。  
> 本文只回答一件事：**按什么顺序做，返工最少。**

排序原则：按**依赖 + 风险**排，不按功能清单排。核心判断——先把两个地基抽象（模板派发契约、语义锚点）锁死，再动生成逻辑，避免重跑两次。

---

## Phase 0 · 地基与止血（动生成逻辑之前）

> 目标：把"以后所有生成都要遵守"的契约先定死，只此一次。这一步纯设计 + 少量代码，不烧生成成本。

1. [x] **删讨论区 + 延伸** —— 页面、入口、发帖接口、服务端读写、seed 与生成链路均已退出；用户想法保留为独立本机记录。
2. [x] **定「模板派发契约」** —— 已落 `{template, confidence, downgrade, renderData}`；前端按 `template` 派发；Phase 0 只启用论证类，其他类型恒定降级论证类。
3. [x] **定「语义锚点」** —— 已落 `anchorId = hash(节点核心命题归一化文本)`；新节点、回响与用户记录均使用锚点。
4. [x] **存量打标迁移** —— 数据库初始化时原地给现有 `analyses` 补 `template:"argument"`、`renderData` 与节点/回响锚点，不重跑内容。

Phase 0 实现基线：`web/lib/analysis-contract.ts`、`web/lib/semantic-anchor.ts`、`web/lib/server/store.ts`、`web/components/reader/AnalysisRenderer.tsx`。

**为什么 2、3 必须最先**：模板和用户记录都长在这两个抽象上。不先定，每加一个模板都在改数据形状、锚点跟着漂，必然重跑两次。锁一次，后面只加不改。

---

## Phase 1 · 单视频五模板（最大的一块，分三步）

**1a — 只跑通「派发脚手架 + 论证类」**  
论证类 = 现有脉络，视觉零变化，但底层走新派发/降级链路。先证明**及格线 + 安全网**端到端能用：任意视频至少稳定出论证类，评委随手点也不白屏。

**1b — 一次加一个模板，各带「硬字段降级判据」**（按价值/风险排序）：

| 顺序 | 模板 | 理由 |
|---|---|---|
| ① 历史类（时间轴） | 真实语料历史/叙事占比最大，回报最高 |
| ② 数据类（图表） | "图替长文"最直观；硬判据清晰（无来源/单位→降级） |
| ③ 对比类（双行） | 结构简单但真实语料少，优先级往后 |
| ④ 条件推演（沙盘） | 最难、幻觉风险最高、可靠条件最少，压轴；做不好就降级论证类 |

降级判据用**必需字段有无**做硬判断，不用玄学置信度数字（可测、不随重跑漂移）。

**1c — 顺手并进去两件事**（不单列阶段）：
- **回响改统一锚点接口**：锚点连锚点，与模板无关（PRD §十五 15.3）。加了非论证模板就必须做。
- **补缺融进节点详情区**（延伸已在 Phase 0 删）。

> **Phase 1 收尾 = 闸门「五类固定做法定稿」**。这之前的重跑都**安全**——线上还没有用户记录功能，不存在冲掉记录的问题。约束只是 Phase 0 已把 anchorId 焊进数据，将来记录挂得上。

## Phase 2 · 用户记录 + 我的岛屿（已完成）

- [x] **独立用户记录层**：AI 层与用户层分开存，用 anchorId 挂靠；v1 记录无损迁移，生成内容更新不会覆盖或删除用户记录。
- [x] **重挂与游离记录**：进入视频时登记最新锚点清单；匹配失败进入想法岛「游离记录」，保留旧节点快照并可跳回原视频。
- [x] **积累可视化**：想法岛承接主动记录，足迹岛自动累计看过的视频、探索节点与查看回响；我的岛屿和子页展示累计数。
- [x] **时序红线已接入**：records UI 已上线；之后任何全量重跑前都要先验证锚点重挂，不得绕过游离记录保护。

Phase 2 实现基线：`web/lib/client/journal.ts`、`web/components/reader/FootprintTrack.tsx`、`web/app/me/thoughts/page.tsx`、`web/app/me/footprints/page.tsx`。

- **评论主题热度图**：零耦合、mock 数据、所有模板通用。Phase 0 论证类脚手架一出来就能挂上做，不排进串行阶段。
- **足迹岛自动日志**：可派生自现有数据，可提前于 Phase 2 起步。

## Phase 3 · 合集页强化 + 大陆收尾

- 合集页：跨视频关系质量（绑双方节点 + 来源）、增量"更新建议"、积累计数——复用 1c 的锚点/回响抽象。
- 大陆：删残留讨论/延伸入口，确认自动聚类合集能进现有合集层。

## Phase 4 · 分享卡 —— 挂起

消费形态没想清，不进首版。等前面跑通、看到真实结果再倒推（PRD §十五 15.7）。

## 一句话时序

```
Phase 0 地基（契约+锚点）
   └─ Phase 1 五模板（1a脚手架+论证 → 1b历史/数据/对比/沙盘 → 1c回响锚点/补缺进详情）  ← 闸门：五类定稿
        └─ Phase 2 用户记录+我的岛屿（records UI 上线前锚点必须就位）
             └─ Phase 3 合集页+大陆收尾
                  └─ Phase 4 分享卡（挂起）
   ‖ 并行轨：评论热度图、足迹岛自动日志
```

**下一步**：Phase 1 按 1a → 1b → 1c 继续并行实施；全量重跑前先用想法岛验证锚点重挂与游离记录保护。
