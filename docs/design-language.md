# 回响 · 内页设计语言（v3.2「阅读仪器」）

> 适用范围：hero.html 以外的全部内页。Hero 是沉浸式海报，自成体系，不受本文约束。

## v3.2 修订（2026-07-12，解析页评审裁决落地）

评审出了七项候选改法，只采纳两项，其余五项维持现状并在此定稿成法——**以下五条是有意为之的设计，后续评审不得再按 v3.1 旧条文判它们违规**：

1. **认知拓展呈现定稿**：「已有/补缺/延伸」三节点挂发丝线做选择器（沿用脉络的点线语言，选中实心、未选空心），下方一次只浮现一块素色板。素色板是「真实物件才成面」条款下的面（平色/直角/无边框无阴影），不算卡片。v3.1 第 3 条「标签压顶 + 内容整宽」**对认知拓展块不再适用**，只保留给知识点/合成块。"三组连排直读"的替代方案已否决。
2. **块间节奏定稿**：章节题上间距 30px（实测块间约 48px）。v3.1 第 5 条的「块间 72px」**作废**——640px 窄栏下 48px 已够分章，再大反而松散。
3. **页头类型标签定稿**：「AI 建议 · X类」挂元信息行右端，虚线底、点击循环切换。保持轻量，不升级成弹层菜单（内部机制的透出口，不是核心控件）。
4. **延伸条目展开示能定稿**：行尾 mono +/− 轻符号，不加整行 hover 洗色、不移位加重——刻意克制，不与 18px 宋体问题行抢地位。
5. **单节点脉络保留竖线**：节点再少，时间戳｜墨点｜标题的三列栅格与竖线不变，不做条件隐藏。

采纳的两项（已落地代码）：

- **CTA hover 不再转琥珀**：`.cta` / `.door` hover 改为底线加粗至 2px、墨色不变。暖金红线（§7）从此无 hover 例外——琥珀含 hover 态只属于回响。
- **章节题右翼不挂常驻教学句**：脉络章的「发光的节点里，有你看过的在回响」已删——金点呼吸 + 荧光笔划线 + 行内「✦ 回响」三重信号自解释。

## v3.1 修订（2026-07-11，高密度阅读站点调研结论落地）

1. **字体族收敛到 2+mono**：宋（标题/问题）+ 黑（其余一切）+ mono（只装纯数字/ASCII）。楷体移除。
2. **mono 禁止装中文**——Consolas 等无中文字形，中文会回退 SimSun 产生错位；中文标签一律黑体 500 小字。
3. **窄栏不开空槽**：左槽版式只属于脉络（时间戳+墨点把槽填满）；认知拓展/知识点等块用「标签压顶 + 内容整宽」——标签（已有/补缺/延伸/立场分布，黑体 500 12px）作为小标题行贴在内容正上方（间距 7~8px），整块从页面左缘起笔。标签和内容之间不许出现空白沟（右对齐边注、宽左槽两版都已被否）。
4. **中文最小字号 12px**；元信息行 13px。
5. **密度节奏**：块内 18~26px + 发丝线，块间 72px + 墨线章节题——分组靠间距对比。
6. **webfont 自托管**（npm @fontsource 包），不走公网 CDN。

## 0. 诊断：前两版为什么被判「AI 卡片堆砌」

1. 每个信息单元都套「边框 + 圆角 + 阴影」的盒子 → 模板感。
2. 字号层级平（标题 17px vs 正文 13px），层级靠颜色和边框硬撑。
3. 装饰替代结构：字距 kicker、水印字、偏移阴影，都是把戏不是版式。
4. 单根居中列，没有栅格、没有左右节奏。

## 1. 对标与借鉴

| 对标 | 借什么 |
|---|---|
| mymind | 内容即界面；UI 铬件近乎不可见；平静、画廊感 |
| Readwise Reader | 「阅读仪器」：强字号对比（display 44px : 正文 15px）、~65 字符行长、暖纸荧光笔高亮 |
| Are.na / Linear | 高密度=索引与表格 + 发丝线，绝不逐条套卡片 |

**核心律**：默认没有盒子。分组靠留白与 1px 发丝线；只有当一个单元是真实"物件"时才允许成面。

## 2. VI 锚点

- 站点类型：私人观看史档案（工具型阅读产品），非营销页。
- 词标：「回响」+ ✦。✦ 是全站唯一图形符号。
- 保留色：琥珀 `#A87B24`（文字）/ `#D9A93F`(点) / `rgba(233,196,106,.4)`(荧光笔划线)。**只给回响**，任何非回响元素禁用暖金。
- 情绪：安静、纸感、器物感；密度高但不吵。

```
墨       --ink #1C1A16     --ink-2 #57534A    --ink-3 #9B958A
线       --rule #E6E2D9（发丝）   分节用 1px --ink 实线
回响     --amber #A87B24   --amber-hot #D9A93F   --amber-wash rgba(233,196,106,.4)
辅助语义 深想 #5A6F8C · 补缺 #5D7D68 · 反对 #A34A32 · 认同 #3E6B4C（低饱和、只用于文字）
字体     serif = Noto Serif SC（自托管 webfont；展示、问题、章节题）
        sans  = Noto Sans SC（自托管 webfont；正文、标签、元信息——其余一切）
        mono  = ui-monospace/Consolas + 黑体兜底（⚠ 只装纯数字/ASCII：时间戳、编号、时长；中文禁止进 mono）

圆角     0（无卡片即无圆角）    阴影：无
动效     页面 240ms 淡入；节点展开 max-height；回响点 3.2s 呼吸。仅此三处。
```

## 4. 版式栅格

- **没有任何导航栏**（顶栏、侧栏都不要，用户明确要求页面整洁）。单栏文档：正文 ≤640px 居中，顶部留 44–92px 呼吸。
- 返回 = 文首一行小字 `← 上一层名称`；页面元信息压成标题下一行 mono（创作者 · 时长 · 节点数 · ✦ 回响数）。
- 文档末尾居中一个 mono 小 ✦ 作结（colophon）。

## 5. 字阶（必须拉开）

```
display  serif 400  clamp(27px→36px)/1.5   页面主题（解析页=问题本身）
section  serif 500  20px   章节题（壹 · 脉络），下压 1px 墨线
entry    serif 400  17–19px/1.8   延伸问题 / 补缺问题 / 知识点
body-lg  sans 400  16px/1.75  ink    阅读主体（脉络节点标题）
body     sans 400  14.5px/1.9 ink-2  展开正文、说明
label    sans 500  12px/.05em ink-3  中文标签（kicker、已有/补缺、关系）
data     mono 400  12px tabular ink-3  时间戳、编号、时长（仅 ASCII）
meta     sans 400  13px ink-3   页头元信息行
```

## 6. 组件规则

- **脉络节点**：三列栅格 `时间戳(mono，左对齐) | 墨点 | 标题`，节点间发丝线。默认纯墨点；有回响 → 琥珀呼吸点 + 标题加荧光笔划线。展开无盒子，回响引用只用 2px 琥珀左线 + 琥珀关系句（黑体 500）+ 下划线文字链跳转。
- **继续想**：编号（01 mono）+ 类型小标（往深想/补缺口，mono 冷色）+ 宋体问题行，展开出正文。无卡片。
- **列表（视频/合集）**：表格式行，hover 整行 `--bg-soft` 洗色；序号 mono，时长右对齐 mono。合集名直接上 24px 宋体，配 10px 分类色块。
- **合成知识点**：宋体问题 + mono 立场统计行（✓2 ／ +1 ／ ✗1 各自语义色）+ 正文（强调=琥珀荧光笔划线）+ mono 来源脚注。
- **CTA**：一律文字链——宋体 17px + 1px 墨色底线，hover 底线加粗至 2px、墨色不变（v3.2 起 hover 不得转琥珀）。全站无实心按钮（讨论区发送键除外）。
- **讨论**：锁条压缩成一行琥珀 mono 文字；评论发丝线分隔；输入框只留一根底线。

## 7. 反模式（红线）

- 逐条内容套边框/圆角/阴影盒子。
- 暖金出现在回响以外的任何地方。
- 同屏两种以上装饰手法（水印字、渐变、纹理……）。
- 层级靠加色不靠字号。
- 实心大按钮、彩色 pill 堆在一起。
