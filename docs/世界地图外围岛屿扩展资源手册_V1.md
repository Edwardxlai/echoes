# 世界地图外围岛屿扩展资源手册 V1

> 适用范围：世界地图页的未开放岛屿、未来区域预留与后续解锁。
>
> 基础资源：沿用 W01–W11 世界地图资源，不重生成现有大陆、海面、三个区域和地标。
>
> 决策优先级：本手册只调整“世界地图能否出现外围岛屿”这一项。与《回响世界地图 · 视觉语言补充 v1.0》中“不增加第四块大陆”冲突时，以本手册为准；经济、历史、科技仍是当前仅有的三个正式内容入口。

## 1. 这次调整要达到什么效果

世界地图继续保留经济、历史、科技三个已开放区域，同时出现少量未开放的外围岛屿。外围岛屿按照现有地图的正式美术标准制作，可以看清完整海岸、地形、建筑和环境细节；未开放状态只改变岛屿的显示方式，不降低资源精度。

本轮不为外围岛屿绑定类别、合集或开放时间，也不提前制作区域内部页面。它们属于已经存在于世界中的地理区域，暂时没有可进入的内容入口。

完成后应满足以下结果：

- 现有世界资源原样保留，不重新生成，也不改变三大区域的海岸线和坐标。
- 新岛屿与现有大陆保持同一相机、光照、材质和细节密度。
- 每座岛屿是独立资源，可以单独移动、隐藏、替换或解锁。
- 锁定状态不需要另一张重新生成的灰色岛屿图。
- 将来决定岛屿用途后，可以在原资源上继续制作，不需要再次改造整个世界地图。

## 2. 实施原则

### 2.1 同精度制作，独立接入

外围岛屿按正式资源制作，不使用剪影、低清占位或只有色块的临时图。这里的“同精度”指相同世界尺度下的可见细节密度一致，包括海岸复杂度、建筑尺寸、地形纹理、阴影精度和水彩笔触，不要求每座岛屿导出成与世界母图相同的像素尺寸。

岛屿必须从透明背景独立导出，再以世界坐标放到现有海面上。禁止把新岛屿和现有大陆重新生成成一张新图，也不要用生成模型对现有世界母图做局部扩图。

### 2.2 资源完整，语义保持开放

岛屿可以做得完整、精致，但在用途确定前少放带有明确类别指向的主地标。基础场景可以包含山地、河流、港湾、普通聚落、道路和植被，这些内容都要达到正式品质。

主地标使用可替换的独立图层。未来岛屿归入新类别时，可以只替换主地标和少量类别装饰，不重做岛体、海岸和普通景观。画面中不能出现空槽、基座或明显等待填充的位置，暂未安装主地标时，那里也要是一处完整的普通景观。

### 2.3 未开放是一种状态

每座外围岛屿只保留一套正式彩色资源。未开放效果由运行时的降饱和、冷灰青薄雾和局部云层完成；静态回退图也从同一份彩色资源合成，禁止重新生成一张机位可能漂移的锁定版本。

锁定状态仍应看得到岛屿的完成度。建议将饱和度保留在正常状态的 35%–55%，亮度保留 75%–88%，再叠加低对比薄雾。不要压成纯黑色块，也不要使用暖金描边。当前产品整体明亮，偏灰青的未探索区域比纯黑遮罩更适合现有视觉语言。

## 3. 单座岛屿的资源包

每座外围岛屿至少交付以下资源：

| 资源 | 用途 | 是否必须 |
|---|---|---|
| 彩色岛体透明图 | 地形、普通建筑、植被和象牙白侧面 | 是 |
| 接触阴影 | 岛体与海面的贴合 | 是 |
| 浅滩层 | 海岸外侧的浅青过渡 | 是 |
| 湿润接触层 | 悬崖与海水交界的窄阴影 | 是 |
| 泡沫层 | 断续浪圈和岸边泡沫 | 是 |
| 高度参考图 | 后续视差、抬升或 3D 改造 | 是 |
| 岛屿范围遮罩 | 锁定处理、选中状态和后续命中区 | 是 |
| 主地标透明图 | 未来类别身份 | 用途确定后再做 |
| 验收合成图 | 与现有世界地图合成检查 | 是 |

彩色岛体、接触阴影、浅滩、湿润层和泡沫层使用同一画布原点，导出后不得分别重新居中。源文件建议按最大显示尺寸的 2 倍制作，最长边不低于 1024 px；运行时再按实际世界尺寸输出 LOD，默认视角下的像素密度不得低于现有 W03 大陆的同尺度区域。

## 4. 岛屿怎么设计

### 4.1 数量与位置

第一批建议制作 2 座外围岛屿，最多 3 座。它们用于说明世界仍向外延伸，不负责填满海面。

- 一座放在当前大陆左侧或左下外海，默认画面能看到约三分之一到二分之一。
- 一座放在右上或右侧外海，通过轻微拖动可以看到完整岛体。
- 岛屿之间不要形成整齐对称的布局，也不要围着现有大陆排成一圈。
- 新岛屿的视觉面积建议为现有单个区域的 35%–65%，避免小得像环境礁石，也不要大到压缩三个现有区域。

当前运行时海面尺寸为 `560 × 560` 世界单位，镜头允许在横向和纵向拖动，外围岛屿可以直接放在现有大陆画布之外。只需增加岛屿图层和坐标，不需要扩大或重新生成海面。

### 4.2 画面要求

外围岛屿继续使用约 30° 正交俯视、左上柔和日光、浅青海洋、克制水彩顶面和薄象牙白侧面。普通道路、屋顶和水纹不得使用暖金，暖金仍只表示回响。

每座岛屿都要有完整的地形关系，例如主山体、次级高地、河流或港湾、聚落和道路。建筑密度可以低于现有三大区域，但单体质量、阴影和材质不能降低。外围岛屿是正式世界资源，不是背景装饰岛。

### 4.3 生成任务的基础提示词

生成时必须提供现有 W01 世界预览和 W03 大陆作为图像参考，并锁定相机与光照。以下提示词作为任务卡基础，不直接省略参考图：

```text
为现有“回响”知识世界地图制作一座可独立合成的正式外围岛屿。岛屿使用与参考世界完全一致的约 30 度正交俯视、左上柔和日光、浅青海洋环境、克制 watercolor gouache 顶面和薄象牙白微缩地形侧面。岛上有完整且精致的海岸、山地、高地、河湾、普通聚落、道路、植被和少量公共建筑，细节密度、建筑尺度、阴影精度与参考大陆一致。整体明亮、安静、轻盈、低饱和，属于同一个世界。岛屿不绑定经济、历史或科技类别，不设置强语义主地标，不出现空槽、文字、UI、粗白边、暖金道路或赛博霓虹。输出单座完整岛屿，周围保留干净色键背景，方便后续制作真透明 Alpha。
```

候选图只比较岛形、地形组织和与现有世界的匹配程度。选定后进入抠图、同坐标技术层制作和合成校正，不能把候选图直接放进前端。

## 5. 生产流程

1. 在现有世界截图上标出岛屿中心点、显示尺寸和默认镜头可见比例。
2. 带 W01、W03 参考图生成 2–4 个候选，记录模型、Prompt、seed 和版本。
3. 选定一张后制作真透明岛体，校正相机角度、左上光向、侧面厚度和色彩。
4. 从同一岛体工程导出接触阴影、浅滩、湿润层、泡沫、高度图和遮罩。
5. 按世界坐标叠加到现有 W01–W11 资源上，分别检查默认视角、拖动视角和移动端视角。
6. 从彩色岛体生成锁定显示参数和静态回退合成，不重新生成锁定岛屿。
7. 输出源文件、运行时 LOD、验收图和回传记录，再进入前端接入。

只有新岛屿资源可以在这条流程中返工。现有大陆出现合成问题时，优先调整新岛屿的颜色、尺寸、位置和遮罩，不修改 W03 及三大区域资源。

## 6. 前端接入方式

### 6.1 数据结构

外围岛屿与当前可进入区域分开配置，建议在世界场景清单中增加 `expansionIslands`，不要提前塞进 `regions`。当前 `WorldRegionId` 和业务数据仍只包含经济、历史、科技。

建议字段：

```ts
interface WorldExpansionIsland {
  id: string;
  position: [number, number, number];
  size: [number, number];
  state: "locked" | "available" | "hidden";
  assets: {
    terrain: string;
    contactShadow: string;
    height: string;
    coast: {
      shallow: string;
      wetContact: string;
      foam: string;
    };
    mask: string;
  };
}
```

`locked` 只展示岛屿和未探索效果，不出现区域标签、数量、回响、悬浮高亮和进入路由；`available` 表示已经补齐业务实体、标签、命中区和区域页面；`hidden` 用于暂时撤下某座岛屿，不删除源资源。

### 6.2 图层顺序

外围岛屿与现有大陆使用相同的世界层级：

```text
水面
远景云
外围岛屿浅滩
外围岛屿湿润接触层
外围岛屿接触阴影
外围岛屿彩色岛体
外围岛屿泡沫
未开放状态层
中景云
前景云
DOM 标签与面板
```

锁定状态层只作用于对应岛屿遮罩，不得降低整张世界地图的饱和度。新岛屿与 W03 大陆的地形层使用相同高度和渲染顺序，避免看起来像浮在页面上的贴纸。

### 6.3 交互

未开放岛屿默认不参与区域交互，也不伪造类别名称。鼠标经过时可以保持无反馈；如果产品需要说明，可在点击后只显示“这片区域尚未开放”，但不展示预计时间和不存在的内容数量。

默认镜头至少露出一座外围岛屿的部分轮廓。用户拖动地图后可以看到完整岛屿，镜头边界仍不能露出海面平面的边缘。现有 `environmentSize` 足够使用，接入时只需核对 `cameraBounds` 和岛屿坐标。

静态回退图和无脚本预览需要同步更新。它们由现有世界预览、岛屿正式资源和锁定状态合成，仍然不重生成世界母图。

## 7. 将来如何解锁

岛屿用途确定后，按以下顺序开放：

1. 确认岛屿对应的类别或新内容实体。
2. 在保留岛体的前提下制作类别主地标和少量语义装饰。
3. 新增区域页面、业务数据、标签、命中区、镜头焦点和路由。
4. 将状态从 `locked` 改为 `available`，移除冷灰青遮罩和局部未探索雾。
5. 更新静态回退图、移动端构图和无障碍入口。

如果未来方向改变，优先保留岛体、地形和海岸，只替换主地标、类别色倾向与少量建筑。岛屿位置需要调整时修改世界坐标，不动现有大陆。

## 9. 验收清单

- [ ] 现有 W01–W11 和三大区域资源没有被重新生成或覆盖。
- [ ] 岛屿在默认显示尺寸下与 W03 的细节密度一致。
- [ ] 相机角度、左上光向、象牙白侧面厚度和投影方向一致。
- [ ] 岛体、阴影、浅滩、湿润层和泡沫共用同一原点。
- [ ] 锁定状态下仍能看到正式资源的地形和建筑细节。
- [ ] 未开放岛屿没有类别名称、内容数量、暖金回响或可进入路由。
- [ ] 默认视角能看到外围岛屿存在，拖动后能看到完整岛体。
- [ ] 桌面端与移动端都不会露出海面边界。
- [ ] 静态回退图与运行时地图包含同一批岛屿。
- [ ] 单座岛屿可以独立隐藏、替换或解锁，不影响现有大陆。

## 10. 停止条件

出现以下情况时停止这次调整，保留当前世界地图：

- 新岛屿必须通过重生成现有大陆才能放入画面。
- 生成工具无法稳定匹配现有相机、光照和建筑尺度。
- 为了显示外围岛屿，需要明显缩小三大区域，导致现有入口难以辨认。
- 移动端只能依靠大幅降低现有地图清晰度才能容纳新资源。

只要岛屿能够作为独立正式资源叠加，这次扩展就不需要改动现有世界主体。
