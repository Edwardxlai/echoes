# 回响地图美术资源生成手册 V2.1

> 版本：V2.1 · 展示级生产版  
> 对应产品：世界地图页 → 区域地图页 → 群岛地图页 → 视频解析页  
> 核心目标：生产一套**可公开展示、可进入应用、可持续扩展、可稳定复用**的地图资产，而不是若干彼此无关的概念插画。  
> 默认视觉：明亮、轻盈、精致的 2.5D 知识世界；浅青海洋、低饱和类别色、轻水彩顶面、薄象牙白微缩模型侧面；暖金色只表示“回响”。

---

# 0. 先看结论

## 0.1 本项目不是“生成几张地图贴上去”

正式地图系统应由以下部分组成：

```text
场景母版工程
+ 基础地形与普通景观
+ 可独立交互对象
+ 前中后景环境层
+ 共享状态效果
+ 低频动效资源
+ 三段页面转场
+ 移动端独立构图
+ LOD 与加载预览
+ SVG 命中区域
+ 相机、标签与状态配置
```

## 0.2 三个层级的正确容量

### 世界层

- 当前固定为 3 个大类：经济、历史、科技。
- 3 个大类没有问题，不需要为了“丰富”继续增加大陆数量。
- 丰富度来自地形、辅助建筑、海洋、云雾、远景和交互状态。

### 区域层

必须区分三个数字：

| 概念 | 含义 | 首发建议 |
|---|---|---:|
| 场景容量 | 一张区域地图最多能自然容纳多少合集 | 每类 6 个 |
| 地标原型库 | 可复用的合集地标造型数量 | 每类先做 4 个，扩展到 6 个 |
| 当前真实入口 | 真实可点击的合集数量 | 严格等于产品里的真实合集 |

不能因为场景容量是 6，就虚构 6 个可点击合集。没有真实内容的位置应是完整普通景观，不是空洞，也不能伪装成入口。

### 群岛层

- 每个真实视频对应 1 座可点击视频岛。
- 少量视频时使用环境礁石、远景岛、浮标、云层和建筑天际线填充画面。
- 大量视频时按海域或主题子群拆分，禁止无限缩小岛屿。
- 支持 1–20+ 条视频；20 条以上使用扩展海域、分段加载或子群入口。

## 0.3 本手册的正式执行方式

生产顺序不再是“先把世界全部做完，再把区域全部做完，再做群岛”，而是：

```text
先完成一条完整纵向切片
世界 → 经济区域 → 看懂通胀群岛 → 一座视频岛 → 解析页 → 返回
```

这条切片在桌面端、移动端、状态、转场、性能全部通过后，再复制和扩展到历史、科技以及其他合集。

# 1. 文件与任务类型

每个任务必须标明类型。

| 标记 | 含义 | 示例 |
|---|---|---|
| `[GEN]` | 可用图像生成工具制作 | 关键帧、地标造型、云层模块 |
| `[EDIT]` | 必须经过抠图、蒙版、合成或 Blender/Figma 后处理 | 透明地标、区域提取、AO |
| `[DESIGN]` | 必须由设计决定，不能让生成模型随机决定 | 点击范围、固定坐标、标签位置 |
| `[CODE]` | 由前端、Canvas、Shader 或动画代码实现 | 水面流动、色彩状态、低频呼吸 |
| `[MASTER]` | 唯一工程母版，后续资源必须从它派生 | Blender、PSB、Figma 工程 |

## 1.1 资源与页面的对应关系

| 页面 | 业务单位 | 主要视觉对象 |
|---|---|---|
| 世界地图页 | 内容大类 | 三个类别区域 |
| 区域地图页 | 合集 | 合集地标 |
| 群岛地图页 | 单条视频 | 视频岛 |
| 视频解析页 | 单条视频内容 | 普通白色内容页面，不继续地图体验 |

# 3. 步骤 0：技术路线小测

## 3.1 目标

在投入正式美术之前，先验证：

- 分层地图是否能稳定运行。
- 目标移动浏览器和 WebView 是否支持。
- 地标抬升、云层视差、水面效果和点击范围是否成立。
- 图片与纹理是否清晰。
- R3F、PixiJS 或混合方案中哪一种最合适。

## 3.2 测试场景资产

| 编号 | 资源 | 类型 |
|---|---|---|
| T00-01 | 1600×1000 浅海与空气底图 | `[GEN]` |
| T00-02 | 一块完整小型大陆 | `[GEN][EDIT]` |
| T00-03 | 一座独立透明地标 | `[GEN][EDIT]` |
| T00-04 | 远景、中景、前景云各 1 个 | `[GEN][EDIT]` |
| T00-05 | 水纹、选中波纹、回响信标测试素材 | `[GEN][EDIT][CODE]` |
| T00-06 | 地标命中区域 SVG | `[DESIGN]` |
| T00-07 | 测试场景配置 JSON | `[DESIGN][CODE]` |

## 3.3 必须验证的操作

- 拖动。
- 有限制缩放。
- 悬浮。
- 第一次点击聚焦。
- 第二次点击进入。
- 返回并恢复原坐标和缩放。
- 云层以不同速度移动。
- 水面低频流动。
- 地标抬升 4–10px 或等效世界单位。
- 选中水波和回响金可以同时存在但不混淆。

## 3.4 技术路线分叉

### 路线 A：R3F / Three.js 混合方案

通过条件：

- 目标移动设备和 WebView 性能可接受。
- 团队可维护 Blender/GLB 资产。
- 镜头推进与真实抬升是核心卖点。

正式交付以以下资源为主：

```text
GLB 地形
GLB 独立地标
碰撞代理
相机预设
灯光预设
PBR/风格化纹理
云雾 Sprite
水面 Shader
DOM 标签
```

### 路线 B：PixiJS / Canvas Sprite 方案

适用条件：

- 移动端兼容性和加载速度优先。
- 团队更擅长分层插画和预渲染。
- 真实 3D 不是必须。

正式交付以以下资源为主：

```text
完整母版
独立区域 PNG
独立地标 PNG
AO 与阴影
深度图
云雾图层
共享状态图集
SVG 热区
```

### 路线 C：混合方案

推荐默认：

- R3F：地图主体、镜头、少量核心地标。
- PixiJS/Canvas：云、水纹、回响、粒子和大量群岛元素。
- DOM：Logo、输入框、文字、数量、信息面板和无障碍。

## 3.5 技术小测通过标准

- 中端移动设备拖动和点击可用。
- 页面没有长时间黑屏。
- 地标实际显示时不糊。
- 触摸命中范围同时满足：
  - 不小于视觉对象 1.5 倍；
  - 不小于约 44×44px 等效尺寸。
- 确认唯一主路线后再进入步骤 1。

# 4. 步骤 1：锁定统一画风

这一步只生成审核图，不直接进入前端。

## 4.1 必须生成的 6 张审核图

| 编号 | 图片 | 核对内容 |
|---|---|---|
| S01 | 三层视觉 DNA 板 | 三层是否属于同一世界 |
| S02 | 世界完整关键帧 | 3 个类别是否清楚 |
| S03 | 区域完整关键帧 | 区域是否丰富且没有空洞 |
| S04 | 少量视频群岛关键帧 | 2–3 条视频是否不空 |
| S05 | 大量视频群岛关键帧 | 15–20 条视频是否不挤 |
| S06 | 环境、状态与转场效果板 | 云、水、回响、选中是否统一 |

## 4.2 全局视觉 DNA

- 明亮、通透、安静、精致。
- 约 30° 正交俯视或极弱透视。
- 左上柔和日光。
- 顶面：克制水彩、水粉和彩铅细节。
- 侧面：薄象牙白微缩模型质感。
- 海洋：浅青蓝，不使用深海蓝。
- 经济：暖麦、浅鼠尾草、浅陶土。
- 历史：灰紫、赭粉、旧石。
- 科技：雾蓝、浅紫、冷灰。
- 暖金：仅用于回响。
- 禁止厚重浮空岩盘、暗黑奇幻、儿童卡通、赛博霓虹、粗白描边和真实航拍。

## 4.3 S01 任务卡

**任务目的**  
验证世界、区域、群岛三个层级的材质、相机、光照和密度是否统一。

**类型**  
`[GEN]`

**参考输入**  
用户批准的明亮 2.5D 参考图。

**输出**  
16:9，至少 3200×1800，PNG。

**Prompt**

```text
制作一张无文字的明亮高端 2.5D 知识地图视觉审核板，同一宽幅画面中自然展示三个层级：一块连贯知识世界大陆、一张完整且高密度的区域地图、一组拥有主要视频岛与环境礁石的群岛地图。三者统一使用约 30 度正交俯视、左上柔和日光、浅青海洋、薄象牙白微缩地形侧面，顶面带克制的 watercolor gouache 与 colored-pencil 细节。世界层宽阔连贯，区域层道路和建筑更密集，群岛层明显拆分但仍有完整海域环境。画面中出现轻薄远景云、中景云、前景遮挡、水面纹理与少量远景结构，主要对象保持清晰。整体 airy, bright, elegant, editorial, premium, refined light-fantasy product map。没有文字、UI、粗白边和金色普通装饰。
```

**验收**

- 三层属于同一世界。
- 不是房地产沙盘。
- 区域不是空底图。
- 群岛少量内容也能成立。
- 云和水增加层次但不遮挡主体。

# 5. 步骤 2：确定工程母版

## 5.1 为什么必须有母版

不能依靠生成模型分别重画：

- 世界底图。
- 三个类别层。
- 主地标。
- 悬浮状态。
- 移动端版本。

分别重画会导致海岸、视角、光照和坐标漂移。

## 5.2 必须建立的母版

```text
world_master.blend / world_master.psb
region_economy_master.blend / psb
region_history_master.blend / psb
region_technology_master.blend / psb
archipelago_c1_master.blend / psb
island_kit_master.blend / psb
common_effects_master.psb
map_layout.fig
```

## 5.3 母版必须固定的参数

| 参数 | 要求 |
|---|---|
| 相机 | 唯一正交相机，记录 position、rotation、ortho size |
| 光源 | 唯一主光方向、强度和软阴影设置 |
| 坐标原点 | 所有对象使用统一原点 |
| 单位 | 地图、地标、视频岛统一尺度 |
| Pivot | 每个对象明确抬升、缩放和转场中心 |
| 图层命名 | base / object / shadow / atmosphere / state / mask |
| 标签锚点 | 存在 Figma 或 JSON 中，不烘焙文字 |
| 点击代理 | 独立 SVG 或低面数碰撞模型 |
| 导出预设 | 桌面、移动、LOD、Preview |
| 版本记录 | 每次修改保留版本号和来源 |

## 5.4 生成图如何进入母版

正确流程：

```text
AI 生成关键帧
→ 选定造型
→ Blender/Figma/PSB 重建或拆层
→ 固定相机与坐标
→ 原位组装
→ 从同一工程导出所有运行时资源
```

错误流程：

```text
分别生成世界、类别、悬浮、移动端
→ 直接叠加
```

# 6. 步骤 3：完成第一条纵向切片

## 6.1 切片范围

第一条切片固定为：

```text
世界地图页
```

# 7. 步骤 4：性能与资源预算

## 7.1 为什么不能只写“使用 4K”

4096×2560 RGBA 纹理解码后约占 40MiB。多个全画布透明层同时驻留，会迅速占满移动端内存。

## 7.2 建议预算

### 世界首屏

| 项目 | 建议上限 |
|---|---:|
| 首次网络下载 | 3–5MB |
| 空闲预加载后总下载 | 8–12MB |
| 同时驻留大纹理 | 3–5 张 |
| 解码纹理内存 | 100–160MiB |
| 首屏 Hero 可交互 | 1.5 秒内优先显示 |
| 地图基本可交互 | 3 秒内，视网络调整 |
| 目标帧率 | 桌面 60fps；中端移动不低于 30fps |

### 单张区域或群岛

| 项目 | 建议上限 |
|---|---:|
| 场景基础下载 | 4–8MB |
| 完整场景含地标 | 10–18MB |
| 转场预加载 | 2–5MB |
| 大纹理同时驻留 | 4–7 张 |
| 环境粒子和道具 | 优先共享图集 |

## 7.3 优化规则

- 全画布状态遮罩尽量使用单通道灰度图。
- Hover、Unseen、New、Echo 优先共享 Shader、图集和参数。
- 云、波纹、船只和鸟群使用小型图集。
- 完整 4K PNG 只保存在源工程中。
- 运行时使用 WebP、AVIF 或 KTX2。
- 转场先加载 Preview，再替换为高清。
- 离开页面后释放不再需要的纹理。
- 不同时保留世界、区域、群岛的全套高清资源。

# 8. 三个层级的状态不能完全相同

## 8.1 世界大类状态

| 状态 | 含义 | 建议表现 |
|---|---|---|
| Default | 普通状态 | 无额外效果 |
| Hover | 指针经过 | 轻微提亮和主地标抬升 |
| Selected | 准备进入 | 区域抬升、冷白海岸反馈 |
| Content Level | 内容积累程度 | 地貌或辅助建筑密度变化 |
| Recent Update | 该类最近新增 | 非金色局部生长痕迹 |
| Echo | 该类出现回响 | 暖金信标和路径 |

世界层不使用“整个大类未观看”作为核心状态，因为大类内部通常同时包含已看与未看内容。

## 8.2 区域合集状态

| 状态 | 含义 | 建议表现 |
|---|---|---|
| Default | 普通合集 | 无额外效果 |
| Hover | 悬浮 | 地标轻抬 |
| Selected | 准备进入 | 地标抬升、道路聚焦 |
| Never Entered | 从未进入合集 | 低对比和非常轻的薄雾 |
| Recent Update | 合集有新视频 | 非金色搭建/生长 |
| Echo | 合集出现回响 | 暖金信标 |

## 8.3 视频岛状态

| 状态 | 含义 | 建议表现 |
|---|---|---|
| Watched | 已观看 | 正常 |
| Unseen | 未观看 | 轻雾和低对比 |
| Hover | 悬浮 | 轻抬和阴影 |
| Selected | 准备进入解析页 | 海岸反馈和水波 |
| New | 最近新增 | 非金色生长痕迹 |
| Echo | 有回响 | 暖金信标与扩散环 |
| Rich | 内容丰富 | 岛体面积与建筑密度更高 |

## 8.4 共享效果与对象专属文件

### 每个对象必须独立

```text
visual
shadow/AO
hitarea
pivot/config
optional local coast mask
thumbnail
```

### 全站共享

```text
hover light atlas
unseen fog atlas
new growth atlas
echo gold atlas
selected ripple atlas
cloud atlas
water atlas
```

不要给每个类别、合集和视频岛都重复制作完整 Hover、Unseen、New、Echo PNG。

# 11. 步骤 7：建立视频岛模块库

## 11.1 为什么使用模块库

逐条从零生图会导致：

- 画风漂移。
- 相机漂移。
- 尺度不一致。
- 无法支撑几十条视频。
- 后续新增成本过高。

采用“固定模块库 + 人工策展组合 + 必要时局部定制”。

## 11.2 每类模块数量

### 地形轮廓

每类至少 8 个：

- 圆形台地。
- 长条半岛。
- 双湾岛。
- 阶梯岛。
- 环形岛。
- 双核心岛。
- 裂谷岛。
- 多层高地岛。

### 建筑组合

每类至少 10 组：

- 主建筑。
- 中型建筑群。
- 小型聚落。
- 港口。
- 桥梁。
- 广场。
- 类别专属设施。
- 低密度装饰建筑。

### 环境模块

- 海岸和浅滩 6 组。
- 植被 4 组。
- 道路 4 组。
- 船只和浮标 4 组。
- 礁石 4 组。
- 远景小岛 3 组。

## 11.3 单座视频岛正式文件

```text
island_[videoId]_visual
island_[videoId]_shadow
island_[videoId]_local_coast_mask
island_[videoId]_hitarea
island_[videoId]_thumbnail
island_[videoId]_config
```

状态由共享图集和代码组合。

# 12. 步骤 8：组装群岛场景

## 12.1 不同视频数量的结构

| 视频数量 | 结构 |
|---:|---|
| 1–3 | 一个中心海域，少量主要岛，环境礁石补完整度 |
| 4–6 | 疏朗三角、环形或弧形 |
| 7–12 | 两个主题海域 |
| 13–20 | 三个主题海域或三条航线 |
| 20+ | 扩展海域、子群入口或分段加载 |

## 12.2 环境对象与内容对象的区别

### 视频岛

- 尺寸更大。
- 对比更高。
- 有视觉中心。
- 有标签。
- 有悬浮和点击反馈。

### 环境对象

- 更小、更低、更淡。
- 不出现大型主建筑。
- 没有标签。
- 没有悬浮和点击反馈。
- 主要使用礁石、沙洲、远景岛、浮标和港标。

少视频场景不建议堆 6–10 个拥有建筑中心的“环境岛”，容易误导。优先使用礁石、沙洲和远景结构。

## 12.3 每个合集正式群岛包

```text
collection_[id]_sea_base
collection_[id]_environment
collection_[id]_far_silhouette
collection_[id]_cloud_config
collection_[id]_water_flow
N 个视频岛
N 个 hitarea
普通航线
状态锚点配置
进入解析页遮罩
depth
preview
mobile composition
```

## 12.4 群岛验收

- 少量视频不空。
- 大量视频不挤。
- 环境对象不被误认成入口。
- 不为增加视频而随机重排已有内容。
- 新增内容优先使用预留海域。
- 视频岛在实际尺寸下仍清晰。

# 13. 步骤 9：环境、状态与动效资源

## 13.1 云层

| 层 | 数量 | 动作 |
|---|---:|---|
| 远景云 | 3–5 | 最慢移动 |
| 中景云 | 4–6 | 中速视差 |
| 前景云 | 3–4 | 较快视差和转场遮挡 |
| 薄雾 | 3 | 远景和未观看 |

云应输出透明模块，不为每张场景重复生成一张 4K 云层。

## 13.2 水面

- 无缝水面纹理。
- 水面法线。
- 低频流向遮罩。
- 选中扩散环。
- 海岸泡沫 Alpha。
- 轻微反射或亮度变化由 Shader 实现。

## 13.3 低频环境动画

- 船只缓慢移动。
- 鸟群偶尔经过。
- 风车缓慢旋转。
- 吊机或观测装置轻微动作。
- 普通航线流动。
- 不使用大幅循环动画干扰阅读。

## 13.4 暖金回响

唯一高饱和状态：

- 小型信标。
- 细光路。
- 呼吸晕。
- 水面扩散环。
- 不使用奖励爆炸、宝箱、星星雨或强光柱。

# 14. 步骤 10：三段转场

## 14.1 世界 → 区域

```text
目标类别提亮和抬升
→ 镜头靠近
→ 前景云从边缘进入
→ 类别轮廓放大
→ 区域 Preview 接管
→ 高清区域资源补齐
```

需要：

- 类别独立视觉层。
- 主地标。
- 区域轮廓参考。
- 云雾遮罩。
- 区域 Preview。
- 标题锚点。

## 14.2 区域 → 群岛

```text
合集地标选中
→ 道路、河流或光路引导
→ 地标放大
→ 地表过渡到浅海
→ 群岛 Preview 接管
```

需要：

- 合集地标。
- 道路/河流引导遮罩。
- 云或地形遮挡。
- 群岛 Preview。
- 水面扩展遮罩。

## 14.3 群岛 → 视频解析页

```text
视频岛抬升
→ 信息面板展开
→ 其他岛降对比
→ 背景按深度轻微虚化
→ 地图淡出
→ 白色解析页接管
```

需要：

- 视频岛独立层。
- 阴影。
- 背景深度图。
- 局部虚化遮罩。
- 解析页首屏预览。

进入解析页后不保留地图拖动、地图背景或地图缩略图。

# 21. 失败重试规则

1. **画面发糊**  
   独立生成或重建主对象；不从全景裁切放大。

2. **区域出现空洞**  
   取消预留圆盘；先完成普通景观，再用地标替换。

3. **地标像贴纸**  
   回到母版原位校正相机、光照、笔触密度和接触阴影。

4. **少视频群岛太空**  
   增加礁石、沙洲、远景结构、云和港标，不虚构可点击内容。

5. **群岛太挤**  
   拆分海域，不继续缩小主要岛。

6. **环境对象像入口**  
   降低尺寸、对比和建筑中心，取消标签与悬浮反馈。

7. **场景僵硬**  
   增加前中后景、低频水面、船只、鸟群和可旋转部件。

8. **状态资产爆炸**  
   改为共享图集、Shader 和参数，不为每个对象复制一套。

9. **科技变赛博**  
   使用 quiet engineering、research infrastructure、modular collaboration。

10. **历史变战争游戏**  
    使用 archive、excavation、layered evidence、reinterpretation。

11. **经济变生态主题**  
    使用 trade、market、production、distribution、port、city network。

12. **普通元素出现暖金**  
    重做；暖金只属于回响。

# 23. 最终结论

世界层固定为三个大类是合理的，重点是把三个区域做得连贯、清晰且具有环境层次。

区域层不能按照“当前只有几个合集”把画面做成几个孤立对象，也不能为了展示丰富度虚构大量入口。正确方式是：

```text
完整区域场景
+ 6 个自然空间节点容量
+ 与真实业务数量一致的可点击地标
+ 可复用地标原型库
+ 普通完整景观
```

群岛层必须通过模块库、海域组织和共享状态支持大量视频，而不是为每条视频分别生成一整张新地图。

正式生产从一条完整纵向切片开始。只有世界、区域、群岛、解析页、返回路径、移动端、状态、转场和性能全部成立后，才扩展到其他大类和合集。
