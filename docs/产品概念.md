# 回响 · 产品概念与对齐

本文档记录**为什么这么设计**，不只是设计成什么。约束推理链是这个项目的脊椎，也是路演时的弹药。

> 本版为「输入视频」范式定稿。之前的「嵌入 feed 被动悬浮球」方案因与赛制冲突（抖音精选是主动观看、提交形态是"输入视频→处理"）已废弃。关键决议见各节 ✅。

---

## 1. 一句话定位

> 输入一条高信息密度视频，回响帮你把"看过"变成"拥有"：看懂它本身的逻辑（脉络），补上它没讲的背景（补缺），接上你已经看过的内容（回响），再把你引向下一步（延伸/讨论）。内容不再刷完就沉底，而是被接进你已有的知识里。

〔2026-07-17 叙事分层修订〕旧版一句话定位把"让视频之间产生联系"当全产品的核心，结果补缺（连接的是外部背景）和延伸（连接的是人与问题）挂不上叙事。本版分层：**理解是目的，连接是机制**——完整推理见 §2。

---

## 2. 面向谁、解决什么（叙事分层版）

### 2.1 用户

**用户心态：** 看抖音精选优质内容的人——"**我想有收获，但收获留不下来、连不起来**"。他关注了几个知识区博主，认真看完过几十条长视频，每条看时都觉得"讲得真好"，几周后只剩零星几个词。收藏夹是坟场，AI 总结完还是一条条孤立的纪要。他不缺获取，缺的是**留存和连接**。

### 2.2 痛点（定位的锚）

> **一条视频是知识的切片，被剪下来时四周都是断口。** 它和自己需要的背景是断的（承重的地方一笔带过）；和你已经看过的一切是断的（内容之间从不互相说话）；和你看完后的下一步也是断的（"被看到"之后价值就停住了）。

**我们解决的仍是"断裂"，但断裂不只发生在视频与视频之间**——旧版痛点锚"内容之间的断裂"是它的子集。用户粘一条抖音链接（或一个合集）过来拿结果，回响把这条视频的断口逐一接上。**这正对应赛题方向一。**

### 2.3 目的与机制分层 ✅

**北极星目的：让用户真正理解并留住一条视频带来的内容——"看过"变成"拥有"。**

连接不是目的，是统一机制：**理解从来不发生在一条视频内部，而发生在这条视频和其它东西接上的地方。** 产品的每个功能都在接一个断口：

| 断口 | 用户的问题 | 功能 |
|------|-----------|------|
| 它自身的逻辑 | 我真的看懂了吗？ | 脉络（时间线 → 论证结构） |
| 它欠你的背景 | 我漏了什么？ | 补缺（戳破承重空洞 + 补外部背景） |
| 你已有的知识 | 它跟我知道的什么关系？ | 回响 + 合集跨视频合成（印证/对撞/互补，顺带完成复习） |
| 下一步 | 然后呢？ | 延伸 + 同题讨论 |

**路演讲法（三问收束）：**

> 看完一条好视频，你其实有三个问题：
> **"我真的看懂了吗？"** —— 脉络 + 补缺
> **"它跟我已经知道的有什么关系？"** —— 回响 + 合集合成
> **"然后呢？"** —— 延伸 + 讨论
>
> 章节要点只回答第一个问题的一半。回响把三个都接住。

**两套话，分开讲：**

- **对用户讲目的**（价值主张）：帮你真正理解、留住、长出下一步——覆盖全部功能，谁也不用硬凑。
- **对评委讲差异**（竞争论证）：我们做的每一件事都发生在"视频与外部"的接缝处——补它没讲的背景、连你的观看史、通向讨论——这些章节要点**结构上做不到**（章节要点只复述视频讲了什么，从不告诉你它没讲什么）。§3/§4 的胜利条件与核心洞察全部继续生效，它们就是这第二套话。

旧叙事的毛病是把第二套话（差异化论证）错当成第一套话（价值主张）来讲。分开之后两边都更硬：用户叙事不再漏功能，评委叙事的"跨视频"论证还多了补缺这个援兵。

## 3. 胜利条件 & 评分细则（北极星）✅

**真评委 = 抖音精选的产品经理。** 赛道本质是抖音拿黑客松做产品调研——它想看的是"**这套内容重构能力，能不能长进抖音精选**"。

| 维度 | 权重 | 关注点 |
|------|------|--------|
| 场景与问题洞察 | 15% | 有价值的场景、清楚的用户、值得被转化 |
| AI 能力与产品结合 | 20% | AI 是否参与理解/重构、是否改变内容使用方式 |
| 体验完整性 | 15% | 可被直接体验、形成基本闭环 |
| **用户价值感** | **30%（最重）** | 转化为可理解/可判断/可执行的能力、提升学习/决策/行动效率 |
| 创新性与延展潜力 | 20% | 新的消费/使用方式、扩展空间 |

**结论：产品/价值/体验导向（非技术、非商业化）。**

**真难点：** 抖音自带 AI 总结 + 章节要点，已把"单条视频的理解/导航"层吃掉。**任何只作用在一条视频内的东西，都打不过章节要点。** 差异化必须换维度——做章节和总结**结构上做不到**的事：**跨视频、连进你的个人观看史。**

## 4. 核心洞察

1. **别做"信息的代理"，做"连接的代理"。** 单条信息处理平台赢；"把你刷过的散内容连起来"平台没做。
2. **加法，不是减法。** 只做"让内容多长出点东西"，不叫用户少看/跳过/judge。
3. **唯一独家资产 = 个人观看轨迹。** 视频谁都能扒；"你刷过什么"只有平台有。产品一切价值从这长出来。
4. **单视频层认命，创新全押跨视频。** 单视频再怎么做都≈章节，是必输的仗；章节永远做不到的"跨视频关联"，才是我们赢的地方。

---

## 5. 约束地图：排除了什么

| 排除 | 为什么 |
|------|--------|
| 嵌入 feed 的被动悬浮球 | 与赛制冲突（抖音精选是主动观看、提交形态是输入视频） |
| 游戏 / 强制交互 | 破坏放松，把消费变成干活 |
| 手动囤积式知识库（你存、你整理、你检索） | 没转化、红海 |
| 又一个单视频总结 / 更漂亮的章节 | 撞平台、打不过、红海 |
| 一大段文字 / 卡片堆砌 | 那就退回总结；结果要视觉化（脉络/点云/发光），关联才有地方长 |
| **在单视频层找创新** | **必输的仗——任何单视频重构都≈章节要点** |

## 6. 产品形态

### 6.0 两段式价值曲线（先立这个框）✅

| 阶段 | 靠什么 | 干的活 |
|------|--------|--------|
| **冷启动（单视频）** | 脉络导航 + 爬到判断/表达 | **"现在就有用"**——day-1 价值，吃价值感 30% |
| **用久了（跨视频）** | 发光节点 + 关联 + 图谱合成 | **"越用越离不开、别人抄不了"**——创新 20% + 壁垒 |

**心法：单视频解决"现在就有用"，跨视频解决"越用越离不开"。冷启动靠前者，差异化靠后者——两者不抢戏，是接力。** 每处理一条视频，库自动厚一点，积累是使用的副产品，不是额外成本。

**两条冷启动 on-ramp：**
1. **单视频输入** —— day-1 就有用（脉络 + 判断/表达）。
2. **合集批量导入（更强）** —— 看完一个合集，一次性把整个合集拖进来，**专属图谱当场生成、回响立刻能响**，一步把用户送到"攒够了"，不用一条条攒。也是个好 demo 动作（现场拖一个合集，一张图谱唰地长出来）。**合集本身就是天然的分类边界**，专属图谱 scope 在合集内，连聚类的 AI 都省了。

### 6.1 核心形式（定稿）✅

> **纯结果：核心问题 + 一条瘦脉络（4~7 个概念节点）+ 发光节点。无视频播放、无跳原片。**

- **输入 = 粘贴抖音链接**（单条视频或合集；贴合官方"粘链接→菜谱"的纯结果范式），上传视频文件当兜底。
- **节点 = 一个概念**（可用"它在回答什么问题"来命名），点它 → **展开这个概念的解析内容**（不播放视频）。时间戳当**文字标注**（"视频 5:30 处"），不当跳转。
- **发光 = 这个概念和你观看史有关联。点开 → AI 一句话说清"这条关联具体是什么关系"。**
- **脉络要"瘦"**：4~7 个节点的骨架，别做成枝繁叶茂的大脑图（那就是普通脑图工具的 low）。丰富度在**节点的交互**（展开/发光），不在图的复杂度。
- **库总览做成卡片式三层**（大类卡 → 合集封面卡 → 内容列表，见 §6.4），自动成型、清晰好上手，不是文件夹管理器。

### 6.2 关联：一个机制，一句话，不分类型 ✅

**关联是"机制"，不是"给用户看的东西"。** 这条是不 low 的命门：

- ❌ "这里有 3 条相关视频"（裸关联）= 推荐位/搜索结果，零价值。
- ✅ 一个关联机制 + **AI 一句话说清怎么个相关法**。

用户要的不是"它们相关"这个事实，是"**它们怎么个相关法**"。那句话会自然长成各种样子——都溶进这一句里，**不建类型、不做分类**（分类是系统里最脆的 AI，砍掉）：

- "他跟你看过的《X》**唱反调**"（争议内容 → 对撞）
- "这里一笔带过，你看的《X》**讲透了**"（补充）
- "这段历史你在《X》**见过**"（叙事/史实 → 印证）

**自适应内容类型、零额外成本**：财经自然长出对撞，历史剧自然长出印证。默认统一一种光 + 那句话，不碰颜色分类。

**AI 稳不稳**：关联 = 概念匹配（稳）；一句话 = 描述两个已知片段的关系（轻生成，稳）。**不碰"观点立场硬分类"这种脆活。**

### 6.3 单视频层的定位（认命，但有一点增值）✅

- 单视频的结构化结果 ≈ 章节要点 / 总结，**打不过，就当基础可用**，别指望它惊艳。
- 唯一比章节多的地方：**爬到导航之上**——展开节点给"继续想 / 形成判断 / 讲清楚"（章节只做导航，从不帮你判断或表达）。这是冷启动的一点增值，也吃价值感 30% 的"可判断/可表达"。
- **红线**：单视频的差异化命在"爬到判断/表达"，不在"能跳转"。别做着做着退回一个漂亮的结构化总结。

### 6.4 知识图谱 & 库总览 — 攒够了（卡片式三层）

> ✅ **v-卡片修订**：原"可缩放点云三高度"作废。点云是"炫酷但增加上手成本"的壳——一堆点不点开不知道是什么，库一大就"找半天"、失去使用念头（正是本节 caution 早写下的警告）。**三个高度的进入逻辑不变，承载形态从点云换成卡片**：大类卡 → 合集封面卡 → 内容列表。详见 [设计文档.md](设计文档.md) v3。

库总览做成**卡片式三层**，三个高度分工明确：

| 高度 | 是什么 | 点进去 |
|------|--------|--------|
| **库总览（大类卡）** | 悬空错位的几张大类卡（经济/历史/科技…），带 `✦N回响` 角标 | 大类=纯分组、点开进合集层 |
| **合集（封面卡）** | 抖音自带封面做成卡片，整齐网格 | **合集级跨视频合成**（吃 30% 价值感的主层） |
| **视频（内容列表）** | 内容已少，列表最好扫（名/创作者/时长/回响标记） | **单视频脉络 + 发光节点 + 「这条的回响」卡**（回响主场，见 §6.1） |

**中点进入 = 合集级跨视频合成：**

- 同一知识点的合成（"这个点，你看过的 A、B 这么讲，C 补充 D，E 反对"）。
- **节点 = 知识点**，明星节点挂 **≥2 来源**（单来源=退回单视频总结=撞平台）。
- **溯源**：节点下**标注来源视频**（标题/创作者/时间戳文字），做次要证据——证明合成来自你真实看过的内容，也回答"是不是 AI 瞎编的"。不承担跳原片。别砍。

**卡片保持"清晰导航"，关联不堆在总览上：**

- 关联已有家（**首页回响 feed、卡片上的 `✦N回响` 角标、解析页「这条的回响」卡**），不需要在总览上画横切连线——那会臃肿。跨世界回响就是某张卡的回响角标指向别世界的内容，照样能演、还不脏。
- 逐层进入 = 渐进披露，天然不成毛线球。
- ⚠️ **分类自动、卡片只用来逛。** 让用户新建/命名/拖拽整理 = 退回手动囤积式知识库（§5 已排除）+ demo 膨胀。管理压到最小。

⚠️ **caution：导航层（卡片）是壳，不是价值高潮。** 它负责清晰好上手地把人带进去；真正的"哦！"在里面（发光 / 回响 / 对撞）。**别让 demo 高潮停在逛卡片**——逛是入场，进入才是戏。炫技（封面球/点云）顶多当一个有界的开场爆点，绝不当检索器。

### 6.5 gated 评论区 — 愿景层

- 门槛由观看史决定：在这个领域刷够了才解锁发言，筛掉没上下文的人（微信读书式高信噪比）。
- 社区需临界规模，**两周做不出**，当护城河讲：seed 示例 + roadmap 页。

## 7. 用户价值感 30% 的定位 ✅

细则："可理解 / 可判断 / 可执行；提升学习 / 决策 / 行动效率"（是"或"）。

- **学习效率**：脉络导航 + 跨视频合成让你更快看懂、串起来。
- **决策效率**：关联/对撞（谁一致谁分歧）帮你拍板。
- 不硬凑"执行"（硬塞工具=花架子）。

✅ **Demo 红线**：不能停在"哦连上了"，必须收在**可见的价值 payoff**——用户当场形成了一个理解升级 / 判断 / 决定，并可发起讨论。

## 10. Demo 策略

- **AI 能真做的真做**：关联匹配、脉络提炼、一句话关系、跨视频合成都可真调模型（用最新 Claude 模型）。**只有评论区 mock。**
- **H5 + 虚拟用户 + 预设观看历史库**：先 seed 一批库（=这个虚拟用户在抖音精选的观看史），再输入一条视频触发功能。
- **合集导入当开场动作**：现场拖一个合集进去 → 专属图谱当场生成，既演了冷启动、又是个视觉爆点。
- **输入 = 粘贴抖音链接**（单条或合集），上传文件当兜底。**链接解析不入设计**：demo 用预烘焙（选定视频/合集离线抽好字幕、算好结果、seed 进去，现场走策展好的链接），生产环境靠平台自有视频 + ASR 数据。
- **合集 = 合集链接解析成视频列表 → 逐条批处理 → 汇成一个"世界"**；demo 整个合集预烘焙。
- **一条动线走完五步**：
  1. 输入一条视频 → 出瘦脉络（核心问题 + 概念节点）
  2. 单视频价值：展开节点 → 继续想 / 形成判断（冷启动价值）
  3. 因为库里有关联 → **节点发光 → 点开 = AI 一句话关系**（招牌瞬间）
  4. 点进 → 知识图谱跨视频合成（价值主战场）
  5. 发起讨论（延展/社区）
- ⚠️ **Mock 风险 = 评委信不信你做得出**：发光那一刻挑"关联明显可检测"的素材，别炫技。

## 12. 已对齐的关键决议（速查）

- ✅ 叙事分层（2026-07-17）：理解是目的、连接是机制；四断口（脉络/补缺/回响与合成/延伸与讨论）；路演三问；对用户讲目的、对评委讲差异，两套话分开

- ✅ 真评委 = 抖音 PM；胜利条件 = 产品/价值/体验导向
