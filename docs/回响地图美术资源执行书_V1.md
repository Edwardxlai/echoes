### 1.1 已经在线上的美术（世界层已打通）

2026-07-13 整理后，`web/public/map/world/` 只保留 4 张被 `WorldTerrain.tsx` 实际消费的 WebP：

| 文件 | 尺寸 | 用法 |
|---|---|---|
| `world_master_visual_default_lod1_v01.webp` | 1586×992 | 世界底图，`<Image fill>` 常驻 |
| `world_*_visual_focus_lod1_v01.webp` ×3 | 1586×992 | 悬浮/选中时整幅透明度切换 |

旧的 `world-sea.png` 与三张 `*-region.png` 没有代码引用，已从运行时目录移除；源图和参考图只在 `map-art/` 留档。四张在线 PNG 转为质量 82 的 WebP 后，合计由 9.62 MB 降至 0.78 MB。

关键事实：**世界层没有用"抠图地标叠加"，而是用"同机位全画幅变体 + opacity 交叉淡化"**。
这条管线已被验证可行，区域层直接复用，不要另起炉灶。

### 1.2 `map-art/00_style/reference-t00/` 逐文件判定

所有 T00 文件均为 **RGB、无 alpha 通道**。T00-02、T00-03 里的棋盘格是**画出来的假透明**，
不能直接当独立图层叠放。

| 文件 | 能否使用 | 判定 |
|---|---|---|
| T00-00 场景母版源图 | ✅ 继续用 | 与 world-base 同尺寸同构图（同源）。作为**画风锚点/参考母版**长期有效；作为 LOD0 分辨率不足（要求 4096 级） |
| T00-01 海空底图 | ✅ 已在用 | 即 world-sea 一系，同上 |
| T00-02 陆地视觉层 | ⚠️ 仅作参考 | 假棋盘格，无真 alpha。世界层已改走全画幅方案，用不上它；留作区域层构图参考 |
| T00-03 地标 visual/shadow | ❌ 不直接用 | ① 假透明；② 金顶/金棚饱和度过高，撞"暖金只属于回响"红线。重生成时改暖麦/浅陶土 |
| T00-04 云 far/mid/front | ⚠️ 应急可用 | 无 alpha。浅色底上可用 CSS `mix-blend-mode: multiply/screen` 凑合；正式版必须 [EDIT] 抠图或重做带 alpha 版本 |
| T00-05 水纹/波纹遮罩 | ✅ 可用 | 作为纹理/遮罩素材给 CSS/Canvas 用，本来就不需要透明 |

**结论：t00 作为画风基准和世界层来源已经"毕业"了；作为分层素材库不合格（无真 alpha、分辨率低）。**
整目录归档为参考资料，不再往 05_t00 里加正式资产。

### 1.3 代码侧真正的缺口

- 区域页（`RegionTerrain.tsx`）和群岛页（`ArchipelagoTerrain.tsx`）目前是**手写 SVG 程序化地形**，
  `map-config.ts` 里 asset 字段是 `"landmark/city"` 这类占位字符串。
- 真实业务量：经济区 2 个合集（c1 通胀 ×3 视频、c2 ×2），历史区 1 个（c3 ×2），科技区 1 个（c4 ×2）。
  共 9 条视频。**当前不存在 15–20 岛的场景，手册 S05 大群岛审核图可延后。**
- `map-config.ts` 的结构（position / labelAnchor / cameraTarget / hitPath / hitBox）与手册 §17 配置规范
  基本一致，无需改造，新资源只需把 asset 从占位串换成真实路径。

## 2. 手册 V2.1 的评价：总体成立，5 处按实际修正

手册的核心原则全部保留：纵向切片优先、母版派生、真实入口≠场景容量、共享状态图集、
暖金只属回响、环境对象不伪装入口。以下 5 点按现状修正：

1. **步骤 0（技术路线小测）已事实完成，跳过。**
   现网世界页 = 预渲染分层 + DOM/CSS + hitPath 命中 + 拖拽缩放，即手册"路线 B 的 DOM 变体"。
   路线 A（R3F/Blender/GLB）对当前团队与产品是过度设计，封存不做。

2. **§5.4"错误流程"需要加一条例外。**
   手册禁止"分别生成→直接叠加"，但世界层实践证明：**同一母版图上局部重绘（inpaint/img2img）
   导出同机位全画幅变体**是允许且推荐的，它天然不需要抠图、不存在坐标漂移。
   区域层 hover 态优先走这条路，而不是抠透明地标。

3. **所有标"透明"的 [GEN] 任务必须强制跟 [EDIT] 抠图工序。**
   生成模型不会输出真 alpha（T00-02/03 的假棋盘格就是教训）。
   任务卡中"透明要求"一栏一律写"生成后由 [EDIT] 抠图产出，不接受棋盘格底"。

4. **母版从简：不建 Blender 工程。**
   母版 = ①每场景一张定稿母版生成图（源 PNG 存 map-art）+ ②PSD/Figma 拆层工程（需要独立层时）
   + ③map-config.ts 即配置母版。手册 §5.2 列的 8 个 .blend/.psb 裁剪为此三件套。

5. **分辨率与格式双轨。**
   手册要求 4096×2560 源图是对的（现有 1586×992 放大到 1360px 舞台已接近极限）；
   但 4K PNG 单张 8–10MB 会爆预算，**运行时一律转 WebP（质量 82 左右），PNG 只留在 map-art 源目录**。
   现有 8 张线上 PNG（合计约 16MB）也应在阶段 0 转 WebP。

### 阶段 6：共享效果、移动端与打包

1. 带真 alpha 的云图集（far/mid/front 各 3–4 个模块，[GEN]+[EDIT] 强制抠图）替换 T00-04。

3. 移动端独立构图核查：只在桌面图确实不可用时才做独立裁切版，不默认重画。

5. 打包核对：map-art 源目录含每张图的 Prompt/模型/seed/版本回传记录。

## 6. 红线（任何阶段违反即打回）

1. 暖金只属于回响；地标、屋顶、航线出现高饱和金即重做（T00-03 前车之鉴）。
2. 交给前端的"透明"素材必须有真 alpha 通道，棋盘格底一律打回。
3. 可点击入口数量 = data.ts 真实业务数量；普通景观不加标签、不加悬浮反馈。
4. 不为单个对象复制整套状态 PNG，状态走共享图集 + CSS/代码。
5. 文字、真实封面、数量徽标不烘进生成图，全部 DOM。
6. 未过阶段 4 验收，禁止启动阶段 5 的任何生图。
