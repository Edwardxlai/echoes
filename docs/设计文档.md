# 回响 · 设计文档

> 本文管 **长什么样、怎么交互、怎么落地**。产品的 **为什么** 见 [产品概念.md](产品概念.md)，冲突时以概念文档的决议为准。
> 选题（历史/文学 vs 经济学）未定，涉及具体内容处用 `〈占位〉` 标注，选题定后填空即可。
>
> **v3 卡片式改版（本次，重大方向调整）**：**放弃点云/星云作为核心导航**。理由：点云是"炫酷但增加上手成本"的东西——一堆点不点开不知道是什么，库一大用户就"找半天"，失去使用念头（这也正是概念文档 §6.4 早已警告的"点云是壳不是价值高潮"）。改为**清晰、低上手成本、可扩展的卡片式层级**：大类卡 → 合集封面卡 → 内容列表 → 解析回响卡。核心不是导航炫技，是**"解析某条视频时，回响以卡片的形式响出来"**。原 v2 的点云/导航栈相关设计作废；push/pop 导航栈保留。

---

## 1. 设计原则

1. **卡片语言贯穿全站**：大类、合集、内容、回响，全部是卡片/列表项——有封面、有名字、有数字、有角标，**不点开就知道是什么**。抖音用户天天刷卡片，零学习成本。
2. **浅层错位、深层规整**：最浅的大类层就几张卡，可以**悬空、错位**排放，取编辑感与格调；越往下卡片越多，**必须回到整齐的网格/列表**，否则错位就成了"乱、难扫"。
3. **回响三处露出**（别让卡片把独家价值埋掉）：
   - **角标**：每张卡右上 `✦ N回响`，连接主动贴到你正在看的东西上，不用去找。
   - **首页回响 feed**：一进首页不点任何东西，就看到"你看过的这些，最近在别处响了"。
   - **解析回响卡**（价值高潮）：把某条视频扔进来/点开时，解析页长出"这条的回响"几张卡。
4. **价值在"进入之后"**：导航层（大类/合集/列表）只负责清晰地把你带到内容；真正的价值（脉络、回响、合成）在钻进去之后。别让任何一屏停在"好看但读不出东西"。
5. **纯结果、无跳原片**：粘链接是来拿结果的，不播放视频、不跳回原片。溯源=标注来源（标题/创作者/时间戳文字），不承担跳转。
6. **只逛不管理**：库自动成型，用户只浏览、进入，不新建/命名/整理分类。
7. **移动端优先**：按 375px 手机屏设计。导航只用手机原生的两件事——点卡片压进详情、左上箭头返回。
8. **克制的壳，让内容发光**：页面 chrome 压成中性冷色，**唯一高饱和的暖金只给"回响"**；合集封面自带的颜色就是内容，让封面跳出来。
9. **点云降级**：不再当家、不当检索器。**顶多作为一个可选的"炫技时刻"**——在内容极少（≤十几条、且每张有封面）时，用一个缓慢转的**封面球**当"进合集"的爆点，或当 demo 开场彩蛋；**绝不用作首页或找东西的工具**。

## 2. 信息架构（卡片三层 + 解析回响）

```
首页（大类层）
  ├─ 顶部：粘贴链接 / 导入合集  ← 主动作
  ├─ 回响 feed：「你看过的，最近在别处响了」几张回响对卡
  └─ 大类卡（悬空错位，少数几张）  经济 / 历史 / 科技…
         │  每张：类名 · N合集 · ✦N回响
         ▼ 点进
      合集层（抖音自带封面做成卡片，整齐网格）
         │  每张：封面 · 合集名 · N条 · ✦N回响
         │  （可选炫技：封面球视图，内容少时用）
         ▼ 点进
      内容层（列表，因为内容已少）
         │  每行：视频名 · 创作者 · 时长 · ✦（有回响）
         ▼ 点进单条 / 或首页直接粘一条视频
      解析页（价值高潮）
         ├─ 核心问题 + 瘦脉络（4~7 概念节点，发光=有回响）
         └─ 「这条的回响」：几张回响卡（具体旧视频 + AI 一句话关系）
                 └─「看全貌」──► 合集合成页 ──「发起讨论」──► 讨论区(gated)

返回：详情页顶部 ← 面包屑，pop 回上一层。
```

三层导航（大类→合集→内容）解决"清晰、好找"，解析回响解决"独家价值"。点云不在主干里。

## 3. 核心流程（= Demo 五步动线）

1. **导入合集**（开场爆点）→ 首页大类里长出/点亮一张新卡（或直接进新合集，封面球炫一下）。
2. **进合集 → 点一条视频**（或首页直接粘一条）→ **解析页**：核心问题 + 瘦脉络。
3. **点节点 → 展开**：继续想 / 形成判断（冷启动价值，单视频层）。
4. **解析页「这条的回响」卡亮起**（招牌瞬间："这条跟你看过的《X》唱反调"），点开=AI 一句话关系。
5. **「看全貌」→ 合集合成 → 发起讨论**（价值主战场 + 延展）。

全程左上 `←` 逐层 pop 回上级卡片页。

## 4. 页面详解（手机屏，375px）

### 首页 · 大类层

```
┌──────────────────────┐
│ 回响                  │  ← 品牌
│ [ 粘贴链接 / 导入合集→]│  ← 顶部主动作
│                      │
│ ✦ 最近的回响          │  ← 回响 feed
│ ┌──────────────────┐ │
│ │《通胀》↔《加息》唱反调│ │
│ └──────────────────┘ │
│                      │
│  ┌───────┐           │  ← 大类卡（悬空错位）
│  │ 经济   │  ┌───────┐│
│  │12合集  │  │ 历史  ││
│  │✦3回响  │  │8合集  ││
│  └───────┘  │✦1回响 ││
│      ┌───────┐───────┘│
│      │ 科技   │        │
│      │✦2回响  │        │
│      └───────┘        │
└──────────────────────┘
```
- 顶部主动作：粘贴单条链接（→解析）/ 导入合集（→合集层新卡）。上传文件作兜底。
- **回响 feed**：几张"回响对卡"（《这条》↔《那条旧的》+ 一句话关系），点开进对应解析/关联。一进来就见独家价值。
- **大类卡**：数量少，**悬空错位**排放取格调。每张=类名 + N合集 + `✦N回响` 角标。点进 → 合集层。

### 合集层 · 封面卡

```
┌──────────────────────┐
│ ← 经济                │  ← 面包屑
│ ┌──────┐ ┌──────┐    │
│ │[封面] │ │[封面] │    │  ← 抖音自带合集封面
│ │看懂通胀│ │宏观30讲│    │
│ │8条·✦3 │ │12条   │    │
│ └──────┘ └──────┘    │
│ ┌──────┐ ┌──────┐    │
│ │[封面] │ │[封面] │    │
│ │货币简史│ │危机复盘│    │
│ │5条·✦1 │ │9条    │    │
│ └──────┘ └──────┘    │
└──────────────────────┘
```
- 整齐网格（卡多了必须规整）。卡片art = **抖音合集封面**；下方 合集名 · N条 · `✦N回响`。
- 点卡 → 内容层。
- **可选炫技（内容少时）**：某合集可切"封面球"视图——十几张封面在缓慢转的球面上，有界、可认、好看，当"进合集"的 wow。仅在内容 ≤ 十几条时用，不当导航。

### 内容层 · 列表

```
┌──────────────────────┐
│ ← 看懂通胀这门课        │
│ ▹ 通胀到底是谁的锅  ✦  │  ← ✦=这条有回响
│   老唐说经济 · 15:20   │
│ ─────────────────    │
│ ▹ 央行加息在加什么     │
│   老唐说经济 · 9:40    │
│ ─────────────────    │
│ ▹ 为什么工资涨了还是变穷 ✦│
│   财经小林 · 12:10     │
└──────────────────────┘
```
- 内容已少，**列表最好扫**：视频名 + 创作者 + 时长 + `✦`（有回响标记）。
- 点行 → 解析页。

### 解析页 · 单视频（价值高潮，纯结果）

```
┌──────────────────────┐
│ ← 看懂通胀›这条        │  ← 面包屑，无播放器
│ 这条在回答：           │
│ 〈核心问题〉           │
│                      │
│ 瘦脉络：              │
│ ● 概念1        1:20  │
│ ● 概念2  ✦     4:05  │ ← 发光=有回响
│ ● 概念3        7:30  │
│ ● 概念4  ✦    11:10  │
│ [ 展开·继续想 ]       │
│                      │
│ ──── 这条的回响 ──── │
│ ┌──────────────────┐ │  ← 回响卡（从发光节点荡出来）
│ │《央行加息在加什么》 │ │
│ │ ✦ 跟这条唱反调     │ │
│ │ 老唐 · 4:05        │ │
│ └──────────────────┘ │
│ ┌──────────────────┐ │
│ │《为什么工资变穷》  │ │
│ │ ✦ 把"代价"讲透了  │ │
│ │ 财经小林 · 8:12    │ │
│ └──────────────────┘ │
└──────────────────────┘
```
- **无播放器**。顶：核心问题锚点。中：瘦脉络 4~7 节点（点节点 → 就地展开解析内容；发光节点=有回响）。
- **下：「这条的回响」= 几张回响卡**（有界，就这条的 3~6 个）。每张 = 具体旧视频名 + AI 一句话关系（唱反调/讲透了/印证）+ 创作者·时间戳（溯源标注，不跳原片）。
- 点回响卡 → 展开完整一句话关系 + `看全貌 →` 合集合成。这就是"回响以卡片形式响出来"。
- 动效点缀：回响卡载入时从对应的发光节点**轻微涟漪/荡出**，主体仍是可读的卡。

### 合集合成 · 合集（价值主战场）

```
┌──────────────────────┐
│ ← 看懂通胀›合成        │
│ 知识点：〈观点〉       │
│  你看过的人里：        │
│  ✔3 认同 ＋1 补充 ✗1 反对│ ←争议型
│  或"这段一笔带过,《X》讲透"│ ←叙事型
│  来源：《A》《B》《C》  │
│  ── 下一个知识点 ──   │
│ [ 发起讨论 → ]        │
└──────────────────────┘
```
- 合集进入后的跨视频合成。节点=知识点，明星节点 ≥2 来源。呈现随内容自适应（争议→立场分布；叙事→印证/补充），都靠 AI 一句话，不建类型。每个知识点带溯源标注。出口：发起讨论 → 讨论区。

### 讨论区 · gated（愿景层）

```
┌──────────────────────┐
│ ← 某话题›讨论区        │
│ 🔒 在这个世界刷够了才能发言│
│  〈seed 评论1〉        │
│  〈seed 评论2〉        │
│  [ 说说你的判断…]（已解锁）│
└──────────────────────┘
```
- 门槛由观看史决定。两周内：seed 几条 + roadmap 讲，不做真社区。

## 5. 关键交互与动效

| 交互 | 行为 | 说明 |
|------|------|------|
| 大类卡 | 悬空错位、轻微视差/浮动 | 仅浅层，取格调 |
| 卡片进入 | 点卡 → push 详情页，镜头轻推淡入 | 大类→合集→内容→解析 一路 push |
| 返回 | 顶部 `← 面包屑`，pop | 移动端标准 |
| 回响角标 | `✦N回响` 微光呼吸 | 全站唯一高亮语言 |
| 回响卡 | 载入时从发光节点涟漪/荡出 | 价值高潮的 signature |
| 展开节点 | 就地展开该概念解析 | 纯结果，不播放视频 |
| 合集导入 | 新卡"亮出来"/新合集封面球转起来 | 开场爆点 |
| 封面球（可选） | 拖动旋转、点封面进解析 | 仅内容少时用，非导航 |

## 6. 视觉方向

**基调：克制的深色壳，让封面与回响发光。**

- **背景/chrome**：近黑带极淡纵深，卡片是略微抬起的深色面（靠背景色阶+柔和阴影分层，不用重边框）。
- **回响 = 唯一的高饱和暖金**：角标 `✦`、回响卡强调、发光节点，全站只有这一种鲜艳色，一眼就等于"连着你看过的东西"。
- **大类用极低饱和的冷色**区分（经济偏青 / 历史偏靛 / 科技偏蓝），绝不与暖金争注意力。
- **合集封面自带颜色 = 内容**：chrome 压中性，让抖音封面跳出来。
- **字体**：核心问题/品牌用衬线（宋体系）取编辑感；卡片标题/UI 用无衬线；创作者/溯源更弱一级。
- **留白**：大量。信息密度靠逐层进入，不靠一屏堆满。
- **动效基调**：慢、稳、呼吸感，避免弹跳闪烁。尊重 `prefers-reduced-motion`。

> 一句话审美目标：干净好上手的卡片流里，某些卡因为你在别处见过，会亮起暖金的回响。

## 7. 组件清单（可复用）

- `CategoryCard`：大类卡（悬空错位，含回响角标）。
- `CollectionCard`：合集卡（抖音封面 + 名 + 条数 + 回响角标）。
- `VideoListItem`：内容层列表项（名/创作者/时长/回响标记）。
- `EchoStrip` / `EchoCard`：解析页「这条的回响」卡，与首页回响 feed 复用同一张卡。
- `Spine` / `GlowNode`：核心问题 + 瘦脉络 + 发光节点（解析页，无播放器）。
- `RelationDetail`：回响卡展开后的 AI 一句话关系 + 溯源 + 看全貌出口。
- `SynthesisView`：合集合成视图。
- `SourceRef`：溯源条目（标注，不跳转）。
- `DiscussionThread`：讨论区（seed）。
- `CoverSphere`（可选）：合集封面球炫技视图，仅内容少时用。
- `NavStack` / `TopBar`：导航栈 push/pop + 面包屑返回。

## 8. Mock 数据结构（demo seed）

```
Category(大类)  { id, name, tint, collections[], echoCount }
Collection(合集) { id, name, cover(抖音封面URL), videos[], synthesis, echoCount }
Video(视频)  { id, title, creator, duration, cover?, coreQuestion, nodes[] }
Node(概念)   { id, label, timestampText, detail, echo? }   // echo存在=发光
Echo(回响)   { targetTitle, creator, timestampText, sentence }  // AI 一句话 + 溯源
Synthesis    { points[{ label, stance/note, sources[] }] }
```
- Demo 至少 seed：**2~3 个 Category**、每个 1~2 个 Collection（带封面）、每 Collection 3~5 条 Video、部分 Node 带 `echo`（发光）。
- 首页回响 feed = 从各 Video 的 `echo` 里挑几条聚合。
- **能真做的用真 AI**（核心问题/脉络/回响一句话/合成），只有讨论区 seed。合集封面用抖音自带图（demo 可用占位图）。

## 9. 技术选型建议（H5）

- 前端：**Web（H5，移动端优先）**，卡片式布局用普通 DOM + flex/grid 即可，不需要 Canvas/WebGL（点云降级后省掉了重渲染）。
- 导航：**单页 + 导航栈（push/pop）**，卡片压进详情、面包屑返回。不用底部 tab、不用点云。
- 可选炫技：封面球用 **CSS 3D transform + DOM `<img>`**（如 SphereImageGrid 类组件，轻量、不碰 three.js），仅"进合集/彩蛋"处用。
- 输入：粘贴链接为主、上传兜底。链接解析不入设计——demo 预烘焙，生产靠平台自有视频 + ASR。
- 无视频播放器，只呈现结果。合集封面 = 抖音自带封面图。
- AI：解析/回响/合成走后端调**最新 Claude 模型**（`claude-opus-4-8` / `claude-sonnet-5`），结果缓存进 seed，保证 demo 稳。
- **原则**：demo 走预设内容路径，稳定压倒通用性。

### 原型（本仓库）

- `prototype/index.html`：自包含单文件 H5 原型，移动端优先，实现本文 v3 卡片式全动线（首页大类 → 合集封面 → 内容列表 → 解析回响卡 → 合成 → 讨论）。手机浏览器直接打开即可点。
- `prototype/galaxy.html`（旧·点云样张）：已降级，仅留作"封面球/炫技"参考，不代表主设计。

## 10. 两周落地范围（深度靠 Demo，别摊薄）

| 必做（做扎实） | 可简化/预渲染 | 只讲不做 |
|---------------|--------------|---------|
| 解析页：核心问题 + 瘦脉络 + **「这条的回响」卡** | 大类/合集卡页（预置 seed） | 真实社区 |
| 合集合成（≥2 来源 + 溯源） | 合集封面球（可选炫技） | 通用分类管理 |
| 一条完整跑通的 demo 五步动线 | 讨论区（seed 几条）、首页回响 feed | 点云总览 |

**红线**：把"解析时的回响"做到无懈可击（价值高潮），卡片导航做到清晰好上手；别把工期烧在炫技（点云/球）上把里子饿死。
